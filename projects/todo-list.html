To-Do List App

Beginner 2-3 hours JavaScript, HTML5, CSS3, localStorage

A complete engineering guide to building a full-featured to-do list application with CRUD operations, drag-and-drop reordering, localStorage persistence, filtering, categories, and due date management.

Project Overview

This to-do list application goes beyond basic CRUD by implementing drag-and-drop reordering, category tagging, due dates with visual warnings, full-text search, and filtered views (all, active, completed, by category). You will learn how to manage application state, persist data with localStorage, implement the HTML5 Drag and Drop API, and build an intuitive user interface with smooth animations. The app is designed mobile-first with touch support for drag-and-drop on mobile devices.

Problem Statement

Task management is a universal need, but existing to-do apps are either too simplistic (single list, no categories) or too complex (full project management suites). Users need a lightweight, customizable task manager that works offline, persists data across sessions, and provides essential features like categorization, due dates, search, and visual priority indicators. The challenge is building an app that feels snappy, works without a server, and handles the complexity of state management with real-time UI updates.

Real World Use Cases

  • Personal Task Management: A student tracks homework assignments with due dates, categorizes by subject, and filters by priority to manage their study schedule.
  • Team Collaboration: A small team uses a shared to-do list (synced via a backend) to track project tasks, assign owners, and monitor progress with status categories (To Do, In Progress, Done).
  • Shopping Planning: A user creates categorized shopping lists (groceries, household items, electronics) with checkboxes and sorts items by aisle location using drag-and-drop.

Industry Relevance

CRUD (Create, Read, Update, Delete) is the most fundamental pattern in software development, used in virtually every application. Mastering CRUD with local state management transfers directly to React/Vue/Angular state management with libraries like Redux, Zustand, or Pinia. The drag-and-drop pattern is used in Kanban boards (Trello, Jira), email clients, and file managers. localStorage patterns are the foundation for offline-first applications and progressive web apps (PWAs).

Architecture Overview

The application uses a Model-View-Controller (MVC) pattern adapted for client-side JavaScript. The Store class manages all application state (tasks, categories, filters) and emits events when state changes. The View layer subscribes to store events and updates the DOM accordingly. The Controller handles user input and calls Store methods. Drag-and-drop is handled through a dedicated DnDManager that normalizes mouse and touch events. State persistence is automatic - every store mutation triggers a debounced save to localStorage. The category system uses a separate store with predefined defaults and allows custom creation.

Step 1: Store Class with Event System

Build a centralized store that holds all application state and emits events when data changes. The View layer subscribes to these events to update the UI efficiently.

class Store {
  constructor() {
    this.listeners = new Map();
    this.state = this.loadState();
  }

  getState() {
    return this.state;
  }

  subscribe(event, callback) {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, []);
    }
    this.listeners.get(event).push(callback);
    return () => {
      const callbacks = this.listeners.get(event);
      const index = callbacks.indexOf(callback);
      if (index > -1) callbacks.splice(index, 1);
    };
  }

  emit(event, data) {
    if (this.listeners.has(event)) {
      this.listeners.get(event).forEach(cb => cb(data));
    }
    this.saveState();
  }

  addTask(taskData) {
    const task = {
      id: crypto.randomUUID(),
      title: taskData.title.trim(),
      description: taskData.description?.trim() || '',
      completed: false,
      createdAt: new Date().toISOString(),
      dueDate: taskData.dueDate || null,
      priority: taskData.priority || 'medium',
      category: taskData.category || 'general',
      order: this.state.tasks.length
    };
    this.state.tasks.push(task);
    this.emit('task:added', task);
    this.emit('state:changed', this.state);
    return task;
  }

  updateTask(id, updates) {
    const index = this.state.tasks.findIndex(t => t.id === id);
    if (index === -1) return null;
    const oldTask = { ...this.state.tasks[index] };
    this.state.tasks[index] = { ...this.state.tasks[index], ...updates };
    this.emit('task:updated', { oldTask, newTask: this.state.tasks[index] });
    this.emit('state:changed', this.state);
    return this.state.tasks[index];
  }

  deleteTask(id) {
    const index = this.state.tasks.findIndex(t => t.id === id);
    if (index === -1) return false;
    const deleted = this.state.tasks.splice(index, 1)[0];
    this.emit('task:deleted', deleted);
    this.emit('state:changed', this.state);
    return true;
  }

  toggleTask(id) {
    const task = this.state.tasks.find(t => t.id === id);
    if (!task) return;
    task.completed = !task.completed;
    task.completedAt = task.completed ? new Date().toISOString() : null;
    this.emit('task:toggled', task);
    this.emit('state:changed', this.state);
  }

  reorderTasks(taskId, newIndex) {
    const currentIndex = this.state.tasks.findIndex(t => t.id === taskId);
    if (currentIndex === -1) return;
    const [task] = this.state.tasks.splice(currentIndex, 1);
    this.state.tasks.splice(newIndex, 0, task);
    this.state.tasks.forEach((t, i) => t.order = i);
    this.emit('task:reordered', this.state.tasks);
    this.emit('state:changed', this.state);
  }

  getFilteredTasks(filters = {}) {
    let tasks = [...this.state.tasks];
    if (filters.search) {
      const q = filters.search.toLowerCase();
      tasks = tasks.filter(t => t.title.toLowerCase().includes(q) || t.description.toLowerCase().includes(q));
    }
    if (filters.status === 'active') tasks = tasks.filter(t => !t.completed);
    if (filters.status === 'completed') tasks = tasks.filter(t => t.completed);
    if (filters.status === 'overdue') tasks = tasks.filter(t => !t.completed && t.dueDate && new Date(t.dueDate) < new Date());
    if (filters.category) tasks = tasks.filter(t => t.category === filters.category);
    if (filters.priority) tasks = tasks.filter(t => t.priority === filters.priority);
    return tasks.sort((a, b) => a.order - b.order);
  }

  loadState() {
    try {
      const saved = localStorage.getItem('todo-state');
      if (saved) {
        const parsed = JSON.parse(saved);
        if (parsed && typeof parsed === 'object' && Array.isArray(parsed.tasks)) {
          return parsed;
        }
      }
    } catch (e) {
      console.warn('Failed to load state, starting fresh:', e);
    }
    return { tasks: [], categories: ['general', 'work', 'personal', 'shopping', 'health'] };
  }

  saveState() {
    try {
      localStorage.setItem('todo-state', JSON.stringify(this.state));
    } catch (e) {
      if (e.name === 'QuotaExceededError') {
        console.warn('localStorage quota exceeded');
      }
    }
  }

  clearCompleted() {
    const completed = this.state.tasks.filter(t => t.completed);
    this.state.tasks = this.state.tasks.filter(t => !t.completed);
    this.emit('tasks:cleared', completed);
    this.emit('state:changed', this.state);
  }
}

Step 2: Drag and Drop with Touch Support

Implement a unified drag-and-drop manager that works with both mouse and touch events. The HTML5 Drag and Drop API needs touch event polyfilling for mobile browsers.

class DnDManager {
  constructor(container, onDrop) {
    this.container = container;
    this.onDrop = onDrop;
    this.dragItem = null;
    this.dragGhost = null;
    this.init();
  }

  init() {
    this.container.addEventListener('dragstart', this.handleDragStart.bind(this));
    this.container.addEventListener('dragover', this.handleDragOver.bind(this));
    this.container.addEventListener('dragend', this.handleDragEnd.bind(this));
    this.container.addEventListener('drop', this.handleDrop.bind(this));
    this.container.addEventListener('touchstart', this.handleTouchStart.bind(this), { passive: false });
    this.container.addEventListener('touchmove', this.handleTouchMove.bind(this), { passive: false });
    this.container.addEventListener('touchend', this.handleTouchEnd.bind(this), { passive: false });
  }

  handleDragStart(e) {
    const item = e.target.closest('.todo-item');
    if (!item) return;
    this.dragItem = item;
    item.classList.add('dragging');
    e.dataTransfer.effectAllowed = 'move';
    e.dataTransfer.setData('text/plain', item.dataset.id);
  }

  handleDragOver(e) {
    e.preventDefault();
    e.dataTransfer.dropEffect = 'move';
    const target = e.target.closest('.todo-item');
    if (!target || target === this.dragItem) return;
    const rect = target.getBoundingClientRect();
    const midY = rect.top + rect.height / 2;
    const placeholder = this.container.querySelector('.drop-placeholder');
    if (e.clientY < midY) {
      target.parentNode.insertBefore(placeholder, target);
    } else {
      target.parentNode.insertBefore(placeholder, target.nextSibling);
    }
  }

  handleDrop(e) {
    e.preventDefault();
    const id = e.dataTransfer.getData('text/plain');
    const placeholder = this.container.querySelector('.drop-placeholder');
    if (placeholder && id) {
      const newIndex = Array.from(this.container.children).indexOf(placeholder);
      placeholder.remove();
      this.onDrop(id, newIndex);
    }
    this.clearDragState();
  }

  handleDragEnd() {
    this.clearDragState();
  }

  handleTouchStart(e) {
    const item = e.target.closest('.todo-item');
    if (!item) return;
    this.dragItem = item;
    const touch = e.touches[0];
    this.touchStartY = touch.clientY;
    this.touchOffset = touch.clientY - item.getBoundingClientRect().top;
    this.dragGhost = item.cloneNode(true);
    this.dragGhost.style.position = 'fixed';
    this.dragGhost.style.opacity = '0.8';
    this.dragGhost.style.pointerEvents = 'none';
    this.dragGhost.style.width = item.offsetWidth + 'px';
    this.dragGhost.style.zIndex = '9999';
    this.dragGhost.style.transform = 'rotate(3deg) scale(1.02)';
    document.body.appendChild(this.dragGhost);
    this.updateGhostPosition(touch);
  }

  handleTouchMove(e) {
    if (!this.dragItem) return;
    e.preventDefault();
    const touch = e.touches[0];
    this.updateGhostPosition(touch);
    const target = document.elementFromPoint(touch.clientX, touch.clientY);
    const item = target?.closest('.todo-item');
    if (item && item !== this.dragItem) {
      const rect = item.getBoundingClientRect();
      const midY = rect.top + rect.height / 2;
      const placeholder = this.container.querySelector('.drop-placeholder') || document.createElement('div');
      placeholder.className = 'drop-placeholder';
      placeholder.style.height = this.dragItem.offsetHeight + 'px';
      if (touch.clientY < midY) {
        item.parentNode.insertBefore(placeholder, item);
      } else {
        item.parentNode.insertBefore(placeholder, item.nextSibling);
      }
    }
  }

  handleTouchEnd(e) {
    if (!this.dragItem) return;
    const placeholder = this.container.querySelector('.drop-placeholder');
    if (placeholder) {
      const newIndex = Array.from(this.container.children).indexOf(placeholder);
      placeholder.remove();
      this.onDrop(this.dragItem.dataset.id, newIndex);
    }
    if (this.dragGhost) {
      this.dragGhost.remove();
      this.dragGhost = null;
    }
    this.dragItem.classList.remove('dragging');
    this.dragItem = null;
  }

  updateGhostPosition(touch) {
    if (this.dragGhost) {
      this.dragGhost.style.left = (touch.clientX - this.dragGhost.offsetWidth / 2) + 'px';
      this.dragGhost.style.top = (touch.clientY - this.touchOffset) + 'px';
    }
  }

  clearDragState() {
    if (this.dragItem) {
      this.dragItem.classList.remove('dragging');
      this.dragItem = null;
    }
    const placeholder = this.container.querySelector('.drop-placeholder');
    if (placeholder) placeholder.remove();
  }
}

Step 3: Rendering Tasks with Virtual DOM Approach

Build an efficient rendering function that only updates changed tasks rather than re-rendering the entire list. Use document fragments for batch DOM updates.

function createTaskElement(task) {
  const li = document.createElement('li');
  li.className = `todo-item priority-${task.priority} ${task.completed ? 'completed' : ''}`;
  li.dataset.id = task.id;
  li.draggable = true;
  const dueDate = task.dueDate ? new Date(task.dueDate) : null;
  const isOverdue = dueDate && dueDate < new Date() && !task.completed;
  const isDueSoon = dueDate && !isOverdue && !task.completed && dueDate < new Date(Date.now() + 86400000 * 2);
  li.innerHTML = `
    

      
      
    

    

      
${escapeHtml(task.title)}

      ${task.description ? `
${escapeHtml(task.description)}
` : ''}
      

        ${task.category}
        ${task.priority}
        ${dueDate ? `
           ${formatDate(dueDate)}` : ''}
      

    

    

      
      
    

  `;
  li.querySelector('.edit-btn').addEventListener('click', () => enterEditMode(li, task));
  li.querySelector('.delete-btn').addEventListener('click', () => store.deleteTask(task.id));
  li.querySelector('input[type="checkbox"]').addEventListener('change', () => store.toggleTask(task.id));
  return li;
}

function renderTasks(tasks) {
  const list = document.getElementById('todo-list');
  const fragment = document.createDocumentFragment();
  tasks.forEach(task => fragment.appendChild(createTaskElement(task)));
  list.innerHTML = '';
  list.appendChild(fragment);
  updateCounter(tasks);
}

function escapeHtml(text) {
  const div = document.createElement('div');
  div.textContent = text;
  return div.innerHTML;
}

function formatDate(date) {
  const now = new Date();
  const diff = date - now;
  const days = Math.ceil(diff / (1000 * 60 * 60 * 24));
  if (days < 0) return `${Math.abs(days)} days ago`;
  if (days === 0) return 'Today';
  if (days === 1) return 'Tomorrow';
  if (days < 7) return `${days} days left`;
  return date.toLocaleDateString('en-US', { month: 'short', day: 'numeric' });
}

const filters = {
  search: '',
  status: 'all',
  category: 'all',
  priority: 'all'
};

function applyFilters() {
  const filtered = store.getFilteredTasks(filters);

const debouncedSearch = debounce((value) => {
  filters.search = value;
  applyFilters();
}, 300);

document.getElementById('search-input').addEventListener('input', (e) => {

});

document.querySelectorAll('.filter-btn').forEach(btn => {

Common Mistakes

1. Corrupted localStorage Data
If the JSON structure in localStorage gets corrupted (e.g., from a previous version of the app), JSON.parse will throw and all data will be lost. Always wrap parsing in a try/catch, validate the parsed structure, and provide a fallback to default state. Version your localStorage key (e.g., 'todo-state-v2') to handle schema migrations.
2. Drag and Drop Touch Event Conflicts
The HTML5 Drag and Drop API doesn't support touch events natively on mobile browsers. Without touch event handling, drag-and-drop silently fails on iPads and iPhones. Implement touch event polyfilling with touchstart, touchmove, touchend, and use preventDefault() to avoid scrolling while dragging.
3. Performance Issues with Large Lists
Re-rendering the entire task list on every state change causes noticeable lag with 100+ tasks. Use a smarter rendering approach: track individual task changes and only update the affected DOM nodes. Use document.createDocumentFragment() for batch inserts. For very large lists, consider virtual scrolling with a windowing technique.
4. Missing Undo/Redo for Destructive Actions
Deleting a task without undo capability leads to data loss frustration. Implement a simple action history stack that stores deleted tasks. Show a toast with an "Undo" button that restores the last removed task and its original position in the list.

Interview Questions

Q: Explain the event loop and how it relates to asynchronous operations in JavaScript.
A: The event loop is JavaScript's mechanism for handling asynchronous operations. The call stack executes synchronous code. Async operations (setTimeout, fetch, DOM events) are handled by Web APIs and their callbacks are placed in the task queue (macrotask) or microtask queue. The event loop continuously checks if the call stack is empty, then processes microtasks first, followed by the next macrotask. This is why Promise.then() callbacks execute before setTimeout callbacks - Promises use the microtask queue.
Q: What is event delegation and why is it useful?
A: Event delegation is a pattern where you attach a single event listener to a parent element instead of individual listeners on each child. It works because events bubble up through the DOM tree. Benefits: better memory usage (fewer listeners), works for dynamically added elements, and cleaner code. In our to-do app, we could listen for clicks on the task list container and use event.target to determine which action was clicked, rather than attaching listeners to each task's edit and delete buttons.
Q: How does localStorage work and what are its limitations?
A: localStorage is a key-value store that persists data across browser sessions. Data is stored as strings (use JSON.stringify/parse for objects). Limitations: 5-10MB storage limit per origin, synchronous API (blocks the main thread), only stores strings, no built-in expiration, not accessible by service workers (use Cache API instead), and data is not encrypted. For larger data, use IndexedDB. For sensitive data, never use localStorage.
Q: Describe the differences between a class and a factory function in JavaScript.
A: Both create objects but have different patterns. Classes use the `new` keyword, support inheritance via `extends`, and define methods on the prototype (memory efficient). Factory functions are regular functions that return an object, don't require `new`, support closures for private variables, and are more flexible for composition over inheritance. Classes are better for large-scale OOP patterns; factory functions are better for simpler cases and when you need true private state.
Q: How would you implement undo/redo functionality in a web application?
A: Implement a command pattern with two stacks: undoStack and redoStack. Each action (add, delete, update, reorder) creates a command object with execute() and undo() methods. Performing an action pushes to undoStack and clears redoStack. Undo pops from undoStack, calls undo(), and pushes to redoStack. For the to-do app, the delete command stores the deleted task and its index for restoration. The command pattern also enables action grouping (e.g., "Clear completed" as a single undoable action).
