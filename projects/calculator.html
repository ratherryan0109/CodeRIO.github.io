Calculator App

Beginner 2-3 hours JavaScript, HTML5, CSS3

A complete engineering guide to building a fully functional calculator with DOM manipulation, event delegation, keyboard support, expression evaluation, calculation history, and theme toggling.

Project Overview

This calculator project goes far beyond a basic arithmetic tool. You will build a scientific-grade calculator featuring a multi-line expression display, operator precedence parsing, keyboard shortcuts, an interactive calculation history panel, and a theme toggle. You will learn advanced DOM manipulation techniques, event delegation patterns, expression parsing algorithms (Shunting Yard), and state management. The calculator supports basic operations, percentages, square roots, exponents, parentheses, and memory functions (MC, MR, M+, M-).

Problem Statement

Most calculator tutorials teach building a calculator that evaluates expressions using `eval()`, which is dangerous and insecure. Developers need to understand how to safely parse and evaluate mathematical expressions, handle operator precedence, manage state across operations, and build an intuitive UI that works with both mouse and keyboard input. The challenge is creating a calculator that handles edge cases like division by zero, consecutive operators, decimal precision, and large numbers while providing a smooth, native-app-like user experience.

Real World Use Cases

  • Everyday Calculations: A student uses the calculator for homework, switching between basic and scientific modes for algebra and trigonometry problems.
  • Financial Planning: A user tracks expenses with running totals, uses memory functions to store recurring amounts, and reviews the calculation history at the end of the month.
  • Engineering Estimates: An engineer performs quick unit conversions and formula calculations, using the expression history to verify previous results without re-entering data.

Industry Relevance

Expression parsing is a fundamental computer science concept used in compilers, interpreters, spreadsheet software, and formula editors. The Shunting Yard algorithm you'll implement was developed by Edsger Dijkstra and is still used in modern applications like Excel, Google Sheets, and Wolfram Alpha. Event delegation patterns are critical for building performant web applications with many interactive elements. Keyboard shortcut handling is essential for power users and accessibility compliance.

Architecture Overview

The calculator follows a state machine architecture with three main states: INPUT (user is typing an expression), EVALUATE (expression is being computed), and DISPLAY (result is shown). The core evaluation engine uses the Shunting Yard algorithm: the infix expression is tokenized, converted to Reverse Polish Notation using a stack-based algorithm, then evaluated using another stack. Button clicks are handled through a single event delegation listener on the calculator body, with each button's data-action attribute determining the behavior. Keyboard events are normalized to the same action map. The history is stored as an array of { expression, result, timestamp } objects in localStorage with a maximum of 50 entries.

Implementation Guide

Step 1: Shunting Yard Algorithm Implementation

The Shunting Yard algorithm converts human-readable infix expressions (e.g., "3 + 4 * 2") to postfix notation ("3 4 2 * +") which is easy to evaluate with a stack. This avoids the security risks of eval().

const PRECEDENCE = {
  '+': 2, '-': 2, '*': 3, '/': 3, '%': 3,
  '^': 4, 'sqrt': 4, 'sin': 5, 'cos': 5, 'tan': 5
};

const ASSOCIATIVITY = {
  '+': 'L', '-': 'L', '*': 'L', '/': 'L',
  '%': 'L', '^': 'R', 'sqrt': 'R'
};

function tokenize(expr) {
  const tokens = [];
  let i = 0;
  while (i < expr.length) {
    if (/\s/.test(expr[i])) { i++; continue; }
    if (/[0-9.]/.test(expr[i])) {
      let num = '';
      while (i < expr.length && /[0-9.]/.test(expr[i])) {
        num += expr[i++];
      }
      const n = parseFloat(num);
      if (isNaN(n)) throw new Error('Invalid number: ' + num);
      tokens.push({ type: 'number', value: n, raw: num });
      continue;
    }
    if (/[+\-*/%^()]/.test(expr[i])) {
      if (expr[i] === '-' && (tokens.length === 0 || tokens[tokens.length - 1].type === 'operator' || tokens[tokens.length - 1].value === '(')) {
        tokens.push({ type: 'unary', value: '-' });
      } else {
        tokens.push({ type: 'operator', value: expr[i] });
      }
      i++;
      continue;
    }
    if (/[a-zA-Z]/.test(expr[i])) {
      let name = '';
      while (i < expr.length && /[a-zA-Z]/.test(expr[i])) name += expr[i++];
      if (name === 'pi' || name === 'PI') tokens.push({ type: 'number', value: Math.PI });
      else if (name === 'e' || name === 'E') tokens.push({ type: 'number', value: Math.E });
      else tokens.push({ type: 'function', value: name.toLowerCase() });
      continue;
    }
    throw new Error('Unexpected character: ' + expr[i]);
  }
  return tokens;
}

function shuntingYard(tokens) {
  const output = [];
  const opStack = [];
  for (const token of tokens) {
    if (token.type === 'number') {
      output.push(token);
    } else if (token.type === 'function') {
      opStack.push(token);
    } else if (token.type === 'unary') {
      opStack.push(token);
    } else if (token.value === '(') {
      opStack.push(token);
    } else if (token.value === ')') {
      while (opStack.length > 0 && opStack[opStack.length - 1].value !== '(') {
        output.push(opStack.pop());
      }
      if (opStack.length === 0) throw new Error('Mismatched parentheses');
      opStack.pop();
      if (opStack.length > 0 && opStack[opStack.length - 1].type === 'function') {
        output.push(opStack.pop());
      }
    } else if (token.type === 'operator') {
      while (
        opStack.length > 0 &&
        opStack[opStack.length - 1].type !== 'function' &&
        opStack[opStack.length - 1].value !== '(' &&
        (PRECEDENCE[opStack[opStack.length - 1].value] > PRECEDENCE[token.value] ||
          (PRECEDENCE[opStack[opStack.length - 1].value] === PRECEDENCE[token.value] &&
            ASSOCIATIVITY[token.value] === 'L'))
      ) {
        output.push(opStack.pop());
      }
      opStack.push(token);
    }
  }
  while (opStack.length > 0) {
    const op = opStack.pop();
    if (op.value === '(' || op.value === ')') throw new Error('Mismatched parentheses');
    output.push(op);
  }
  return output;
}

function evaluateRPN(tokens) {
  const stack = [];
  for (const token of tokens) {
    if (token.type === 'number') {
      stack.push(token.value);
    } else if (token.type === 'unary') {
      const a = stack.pop();
      if (a === undefined) throw new Error('Insufficient operands');
      stack.push(-a);
    } else if (token.type === 'function') {
      const a = stack.pop();
      if (a === undefined) throw new Error('Insufficient operands');
      switch (token.value) {
        case 'sqrt': stack.push(Math.sqrt(a)); break;
        case 'sin': stack.push(Math.sin(a)); break;
        case 'cos': stack.push(Math.cos(a)); break;
        case 'tan': stack.push(Math.tan(a)); break;
        case 'abs': stack.push(Math.abs(a)); break;
        case 'round': stack.push(Math.round(a)); break;
        case 'floor': stack.push(Math.floor(a)); break;
        case 'ceil': stack.push(Math.ceil(a)); break;
        default: throw new Error('Unknown function: ' + token.value);
      }
    } else if (token.type === 'operator') {
      const b = stack.pop();
      const a = stack.pop();
      if (a === undefined || b === undefined) throw new Error('Insufficient operands');
      switch (token.value) {
        case '+': stack.push(a + b); break;
        case '-': stack.push(a - b); break;
        case '*': stack.push(a * b); break;
        case '/':
          if (Math.abs(b) < Number.EPSILON) throw new Error('Division by zero');
          stack.push(a / b);
          break;
        case '%': stack.push(a % b); break;
        case '^': stack.push(Math.pow(a, b)); break;
        default: throw new Error('Unknown operator: ' + token.value);
      }
    }
  }
  if (stack.length !== 1) throw new Error('Invalid expression');
  return stack[0];
}

function evaluate(expression) {
  const tokens = tokenize(expression);
  const rpn = shuntingYard(tokens);
  const result = evaluateRPN(rpn);
  if (!isFinite(result)) throw new Error('Result is not finite');
  return result;
}

Use a single event listener on the calculator container to handle all button clicks. Each button's data-action attribute determines the handler.

  sqrt: () => appendFunction('sqrt('),
  square: () => appendOperator('^').then(() => appendNumber(2)),
  reciprocal: () => calculate().then(() => appendOperator('/').then(() => appendNumber(1))),
  history: () => toggleHistory(),
  theme: () => toggleTheme()
};

calculator.addEventListener('click', (e) => {
  e.stopPropagation();
  const button = e.target.closest('.calc-btn');
  if (!button || button.disabled) return;
  const action = button.dataset.action;
  const value = button.dataset.value;
  if (actions[action]) {
    actions[action](value);
    updateDisplay();
    buttonFeedback(button);
  }
});

function buttonFeedback(btn) {
  btn.classList.add('pressed');
  setTimeout(() => btn.classList.remove('pressed'), 100);
}

Step 4: History Panel with localStorage

Build a history panel that stores calculation results with timestamps and allows reusing previous results.

const HISTORY_KEY = 'calc-history';
const MAX_HISTORY = 50;

const History = {
  getAll() {
    try {
      return JSON.parse(localStorage.getItem(HISTORY_KEY)) || [];
    } catch { return []; }
  },

  add(expression, result) {
    const history = this.getAll();
    history.unshift({
      id: Date.now(),
      expression,
      result: formatNumber(result),
      timestamp: new Date().toLocaleString()
    });
    if (history.length > MAX_HISTORY) history.pop();
    localStorage.setItem(HISTORY_KEY, JSON.stringify(history));
    return history;
  },

  clear() {
    localStorage.removeItem(HISTORY_KEY);
  },

  delete(id) {
    const history = this.getAll().filter(h => h.id !== id);
    localStorage.setItem(HISTORY_KEY, JSON.stringify(history));
    return history;
  }
};

function renderHistory() {
  const panel = document.getElementById('history-panel');
  const history = History.getAll();
  if (history.length === 0) {
    panel.innerHTML = '
No calculations yet
';
    return;
  }
  panel.innerHTML = history.map(item => `
    

      
${escapeHtml(item.expression)}

      
= ${item.result}

      

        ${item.timestamp}
        
        
      

    

  `).join('');
  panel.querySelectorAll('.history-reuse').forEach((btn, i) => {
    btn.addEventListener('click', () => {
      const result = history[i].result.replace(/,/g, '');
      setExpression(result);
      closeHistory();
    });
  });
  panel.querySelectorAll('.history-delete').forEach((btn, i) => {
    btn.addEventListener('click', () => {
      History.delete(history[i].id);
      renderHistory();
    });
  });
}

Step 5: Expression Display with Formatting

Build the display system that shows the expression with proper formatting and the current result in a separate line. Handle overflow with font-size reduction.

const state = {
  expression: '',
  result: null,
  memory: 0,
  newExpression: true,
  justEvaluated: false
};

function updateDisplay() {
  const exprEl = document.getElementById('calc-expression');
  const resultEl = document.getElementById('calc-result');
  exprEl.textContent = state.expression || '0';
  if (state.result !== null) {
    resultEl.textContent = '= ' + formatNumber(state.result);
    resultEl.style.display = 'block';
  } else {
    resultEl.style.display = 'none';
  }
  adjustFontSize(exprEl);
  adjustFontSize(resultEl);
}

function adjustFontSize(el) {
  const maxWidth = el.parentElement.clientWidth - 40;
  let fontSize = 2;
  el.style.fontSize = fontSize + 'rem';
  while (el.scrollWidth > maxWidth && fontSize > 0.8) {
    fontSize -= 0.1;
    el.style.fontSize = fontSize + 'rem';
  }
}

function formatNumber(num) {
  if (!isFinite(num)) return 'Error';
  if (Math.abs(num) >= 1e15 || (Math.abs(num) < 1e-10 && num !== 0)) {
    return num.toExponential(6);
  }
  const parts = num.toFixed(10).replace(/\.?0+$/, '').split('.');
  parts[0] = parts[0].replace(/\B(?=(\d{3})+(?!\d))/g, ',');
  return parts.join('.');
}

function appendNumber(value) {
  if (state.justEvaluated) {
    state.expression = '';
    state.result = null;
    state.justEvaluated = false;
  }
  state.expression += value;
  state.result = null;
}

function appendOperator(op) {
  if (state.justEvaluated) {
    state.justEvaluated = false;
  }
  const lastChar = state.expression.slice(-1);
  if (/[+\-*/^]/.test(lastChar)) {
    state.expression = state.expression.slice(0, -1);
  }
  state.expression += op;
  state.result = null;
}

function calculate() {
  try {
    const result = evaluate(state.expression);
    History.add(state.expression, result);
    state.result = result;
    state.expression = formatNumber(result);
    state.justEvaluated = true;
  } catch (error) {
    showError(error.message);
    state.result = null;
  }
}

function clearAll() {
  state.expression = '';
  state.result = null;
  state.justEvaluated = false;
}

function deleteLast() {
  if (state.justEvaluated) {
    clearAll();
    return;
  }
  state.expression = state.expression.slice(0, -1);
  state.result = null;
}

Common Mistakes

1. Using eval() for Expression Evaluation
eval() executes arbitrary JavaScript code, making it a severe security risk. If a user types "2+alert('xss')", eval() will execute the alert. Always use a proper parser like the Shunting Yard algorithm. If you must use eval (e.g., for very complex math), sanitize the input rigorously with a whitelist regex: /^[\d+\-*/().%^,\s]+$/.
2. Floating-Point Precision Issues
0.1 + 0.2 in JavaScript equals 0.30000000000000004 due to IEEE 754 binary representation. Never compare floating-point numbers directly with ===. Use Number.EPSILON for comparisons. For display, use toFixed() or toPrecision() with appropriate rounding. For financial calculations, use integer arithmetic (cents instead of dollars).
3. Operator Precedence and Associativity Mistakes
Many calculator implementations get operator precedence wrong, especially for exponentiation (which is right-associative: 2^3^2 = 512, not 64). Unary operators (negation) should have higher precedence than binary operators. Parentheses must be properly tracked with a counter to detect mismatches. Test with edge cases like consecutive operators, leading operators, and trailing operators.
4. Keyboard Event Conflicts with Browser Shortcuts
Without proper handling, keyboard shortcuts like Ctrl+W (close tab), Ctrl+T (new tab), or browser back/forward (Alt+Left) can interfere with calculator input. Use e.preventDefault() to stop default browser behavior for handled keys. Avoid overriding system-level shortcuts. Check for modifier keys (ctrlKey, metaKey, altKey) before handling key events.

Interview Questions

Q: Explain the Shunting Yard algorithm and its use cases beyond calculators.
A: The Shunting Yard algorithm, developed by Edsger Dijkstra, converts infix notation (human-readable: "3 + 4") to postfix/RPN notation (machine-evaluable: "3 4 +"). It uses two stacks: one for operators and one for output. Operators are moved to the output stack based on precedence and associativity rules. Beyond calculators, it's used in: SQL query parsers, spreadsheet formula engines (Excel), mathematical typesetting (LaTeX), programming language compilers, and any system that needs to evaluate mathematical or logical expressions.
Q: How do you handle floating-point precision in JavaScript?
A: JavaScript uses IEEE 754 double-precision binary representation, causing precision issues with decimal fractions (0.1 + 0.2 !== 0.3). Strategies: use toFixed(n) for display rounding; multiply by a power of 10 to work with integers; use a library like decimal.js for arbitrary precision; use Number.EPSILON for approximate equality checks; for currency, always use integer cents. Never use === for floating-point comparison. Example: Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON.
Q: What is event delegation and why would you use it for a calculator with 30+ buttons?
A: Event delegation means attaching a single event listener to a parent element and using event.target to determine which child triggered it. For a calculator with 30+ buttons, this avoids 30+ separate event listeners, reducing memory usage and setup code. It automatically handles dynamically added buttons. The pattern: parent.addEventListener('click', (e) => { const btn = e.target.closest('.calc-btn'); if (!btn) return; handleAction(btn.dataset.action, btn.dataset.value); }).
Q: How would you implement undo functionality for a calculator?
A: Implement a state history stack that stores snapshots of the calculator state (expression, result, memory) after each action. The undo action pops the last state and restores it. Limit the stack to 50 entries to prevent memory issues. For a simpler approach, maintain an expression history array and a pointer to the current expression index, allowing both undo and redo by moving the pointer forward and backward through history.
Q: Explain the difference between prefix, infix, and postfix notation.
A: Infix: operator between operands (3 + 4), requires parentheses and precedence rules, used in everyday math. Prefix (Polish): operator before operands (+ 3 4), no parentheses needed, used in Lisp dialects. Postfix (Reverse Polish): operator after operands (3 4 +), no parentheses or precedence needed, easy to evaluate with a stack, used in old HP calculators and stack-based virtual machines. RPN evaluation is O(n) with a single pass using a stack, making it efficient for expression evaluation.
