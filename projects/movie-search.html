Movie Search App

Intermediate 6-8 hours TMDB API, JavaScript, Infinite Scroll

A comprehensive engineering guide to building a movie search application with TMDB API integration, debounced search, infinite scroll, favorites, ratings, and trailer modal previews.

Project Overview

This movie search application leverages The Movie Database (TMDB) API to provide a rich movie discovery experience. Users can search for movies with debounced input, browse trending/popular movies, view detailed information including ratings, genres, cast, and runtime, watch trailers in a modal, save favorites to localStorage, and scroll infinitely through search results. The project teaches API integration patterns, performance optimization through debouncing, intersection observer for infinite scroll, state management with localStorage, and responsive card layouts with lazy-loaded images.

Problem Statement

Movie databases contain millions of titles, making it challenging for users to discover content efficiently. Existing movie apps are often slow, cluttered with ads, or require paid subscriptions. Developers need to understand how to work with complex REST APIs (multiple endpoints for search, details, credits, videos), handle API rate limits, implement performant search with debouncing, manage large result sets with infinite scroll, and create an engaging UI with trailers and ratings. The challenge is building an app that feels native, loads quickly, and provides a seamless browsing experience.

Real World Use Cases

  • Movie Night Planning: A user searches for movies by genre, checks ratings and reviews, watches trailers, and saves favorites for movie night decisions.
  • Film Research: A student researches films for a project, browsing by director, cast, or release year, reading overviews and saving relevant titles.
  • Content Discovery: A streaming service aggregator uses the TMDB API to build a cross-platform movie discovery tool showing where each title is available to stream.

Industry Relevance

TMDB is the largest community-built movie database, serving over 10 billion API requests monthly to companies like Netflix, Spotify, and IMDb. The integration patterns you learn apply to any REST API (Spotify, YouTube, GitHub). Debounced search is used by every major search product (Google, Algolia, Elasticsearch). Infinite scroll powers social feeds (Twitter, Instagram, TikTok) and content browsing. Intersection Observer is the standard for lazy loading and scroll-based interactions, supported in all modern browsers.

Architecture Overview

The application follows a service-oriented architecture. The TMDB service layer encapsulates all API calls with caching, rate limiting awareness, and error normalization. The search manager handles debouncing with a 400ms delay, maintains a request queue, and aborts in-flight requests when new searches are issued (using AbortController). The intersection observer triggers page loading when the sentinel element is 200px from the viewport. Movie data flows through a pipeline: raw API response -> data normalizer -> renderer. The normalizer maps TMDB's snake_case fields to camelCase, computes derived values (star ratings from vote_average), and resolves image URLs. Favorites are managed separately with a localStorage-backed store that emits custom events for cross-component communication. Trailer modals use the YouTube IFrame API for embedding with auto-play and responsive sizing.

Step 1: TMDB API Service with Caching

Build a robust API service layer with request caching, rate limit awareness, and AbortController support for cancelable requests.

const TMDB = {
  BASE: 'https://api.themoviedb.org/3',
  IMG: 'https://image.tmdb.org/t/p',
  KEY: 'YOUR_TMDB_API_KEY',
  cache: new Map(),

  async fetch(endpoint, params = {}, signal = null) {
    const query = new URLSearchParams({ api_key: this.KEY, ...params });
    const url = `${this.BASE}${endpoint}?${query}`;
    const cacheKey = url;

    if (this.cache.has(cacheKey)) {
      const cached = this.cache.get(cacheKey);
      if (Date.now() - cached.timestamp < 300000) { // 5 min cache
        return cached.data;
      }
      this.cache.delete(cacheKey);
    }

    try {
      const response = await fetch(url, { signal });
      if (!response.ok) {
        const error = await response.json().catch(() => ({}));
        throw new Error(error.status_message || `HTTP ${response.status}`);
      }
      const data = await response.json();
      this.cache.set(cacheKey, { data, timestamp: Date.now() });
      return data;
    } catch (error) {
      if (error.name === 'AbortError') throw error;
      throw new Error(`TMDB API error: ${error.message}`);
    }
  },

  searchMovies(query, page = 1, signal = null) {
    return this.fetch('/search/movie', { query, page, include_adult: false }, signal);
  },

  getTrending(timeWindow = 'week', page = 1) {
    return this.fetch(`/trending/movie/${timeWindow}`, { page });
  },

  getMovieDetails(id) {
    return this.fetch(`/movie/${id}`, { append_to_response: 'credits,videos,recommendations' });
  },

  getMovieCredits(id) {
    return this.fetch(`/movie/${id}/credits`);
  },

  getMovieVideos(id) {
    return this.fetch(`/movie/${id}/videos`);
  },

  getImageUrl(path, size = 'w500') {
    if (!path) return 'https://via.placeholder.com/300x450?text=No+Poster';
    return `${this.IMG}/${size}${path}`;
  },

  getBackdropUrl(path, size = 'original') {
    if (!path) return null;
    return `${this.IMG}/${size}${path}`;
  },

  getRatingColor(voteAverage) {
    if (voteAverage >= 7) return '#22c55e';
    if (voteAverage >= 5) return '#eab308';
    return '#ef4444';
  },

  getStarRating(voteAverage) {
    const stars = Math.round(voteAverage / 2);
    return '?'.repeat(stars) + '?'.repeat(5 - stars);
  },

  clearCache() { this.cache.clear(); }
};

Step 2: Debounced Search with AbortController

Implement a debounced search that cancels previous in-flight requests when the user continues typing, preventing race conditions and reducing API calls.

class SearchManager {
  constructor(onResult) {
    this.query = '';
    this.page = 1;
    this.totalPages = 0;
    this.onResult = onResult;
    this.abortController = null;
    this.pendingRequest = false;
  }

  debounce(fn, delay = 400) {
    let timeoutId;
    return (...args) => {
      clearTimeout(timeoutId);
      timeoutId = setTimeout(() => fn.apply(this, args), delay);
    };
  }

  async executeSearch(query, page = 1) {
    // Cancel previous request
    if (this.abortController) {
      this.abortController.abort();
    }
    this.abortController = new AbortController();

    if (!query.trim()) {
      this.onResult({ results: [], page: 1, totalPages: 0, query: '' });
      return;
    }

    this.pendingRequest = true;
    showLoadingState();

    try {
      const data = await TMDB.searchMovies(query, page, this.abortController.signal);
      this.page = data.page;
      this.totalPages = data.total_pages;
      this.pendingRequest = false;
      hideLoadingState();
      this.onResult({
        results: data.results.map(normalizeMovie),
        page: data.page,
        totalPages: data.total_pages,
        query,
        totalResults: data.total_results
      });
    } catch (error) {
      if (error.name === 'AbortError') return; // Ignore aborted requests
      this.pendingRequest = false;
      hideLoadingState();
      showError('Failed to search movies. Please try again.');
    }
  }

  search = this.debounce(function(query) {
    this.query = query;
    this.page = 1;
    this.executeSearch(query, 1);
  }, 400);

  loadNextPage() {
    if (this.pendingRequest || this.page >= this.totalPages) return;
    this.executeSearch(this.query, this.page + 1);
  }

  reset() {
    this.query = '';
    this.page = 1;
    this.totalPages = 0;
    if (this.abortController) this.abortController.abort();
  }
}

function normalizeMovie(movie) {
  return {
    id: movie.id,
    title: movie.title,
    overview: movie.overview,
    posterPath: movie.poster_path,
    backdropPath: movie.backdrop_path,
    releaseDate: movie.release_date,
    releaseYear: movie.release_date?.split('-')[0] || 'N/A',
    voteAverage: movie.vote_average,
    voteCount: movie.vote_count,
    genreIds: movie.genre_ids || [],
    popularity: movie.popularity,
    originalLanguage: movie.original_language
  };
}

Step 3: Infinite Scroll with Intersection Observer

Implement infinite scroll using Intersection Observer API that loads more results when the user scrolls near the bottom of the page.

class InfiniteScroll {
  constructor(loadMoreCallback, options = {}) {
    this.loadMore = loadMoreCallback;
    this.options = {
      root: null,
      rootMargin: '200px',
      threshold: 0.1,
      ...options
    };
    this.sentinel = null;
    this.observer = null;
    this.isLoading = false;
    this.hasMore = true;
    this.init();
  }

  init() {
    this.sentinel = document.createElement('div');
    this.sentinel.className = 'scroll-sentinel';
    this.sentinel.innerHTML = '
';
    document.querySelector('.results-grid').after(this.sentinel);

    this.observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting && this.hasMore && !this.isLoading) {
          this.loadMoreContent();
        }
      });
    }, this.options);

    this.observer.observe(this.sentinel);
  }

  async loadMoreContent() {
    this.isLoading = true;
    this.sentinel.querySelector('.scroll-loader').style.display = 'flex';
    try {
      await this.loadMore();
    } catch (error) {
      console.error('Infinite scroll load failed:', error);
    } finally {
      this.isLoading = false;
      this.sentinel.querySelector('.scroll-loader').style.display = 'none';
    }
  }

  setHasMore(hasMore) {
    this.hasMore = hasMore;
    if (!hasMore) {
      this.sentinel.innerHTML = '
You\'ve reached the end
';
    }
  }

  destroy() {
    if (this.observer) {
      this.observer.unobserve(this.sentinel);
      this.observer.disconnect();
    }
    if (this.sentinel) this.sentinel.remove();
  }
}

// Usage with search manager
const infiniteScroll = new InfiniteScroll(async () => {
  if (searchManager.page < searchManager.totalPages) {
    await searchManager.loadNextPage();
  } else {
    infiniteScroll.setHasMore(false);
  }
});

// After each render, check if there are more pages
function renderMovies(result) {
  const grid = document.querySelector('.results-grid');
  if (result.page === 1) grid.innerHTML = '';
  result.results.forEach(movie => grid.appendChild(createMovieCard(movie)));
  infiniteScroll.setHasMore(result.page < result.totalPages);
}

Step 4: Favorites System with localStorage

Implement a favorites management system with CRUD operations, custom events for real-time UI updates, and a dedicated favorites view.

class FavoritesManager {
  constructor() {
    this.storageKey = 'movie-favorites';
    this.favorites = this.load();
  }

  load() {
    try {
      return JSON.parse(localStorage.getItem(this.storageKey)) || [];
    } catch { return []; }
  }

  save() {
    localStorage.setItem(this.storageKey, JSON.stringify(this.favorites));
    this.emitChange();
  }

  add(movie) {
    if (!this.favorites.some(f => f.id === movie.id)) {
      this.favorites.push({
        id: movie.id,
        title: movie.title,
        posterPath: movie.posterPath,
        voteAverage: movie.voteAverage,
        releaseYear: movie.releaseYear,
        addedAt: Date.now()
      });
      this.save();
      return true;
    }
    return false;
  }

  remove(movieId) {
    this.favorites = this.favorites.filter(f => f.id !== movieId);
    this.save();
  }

  isFavorite(movieId) {
    return this.favorites.some(f => f.id === movieId);
  }

  getAll() {
    return this.favorites;
  }

  getCount() {
    return this.favorites.length;
  }

  emitChange() {
    const event = new CustomEvent('favorites:changed', {
      detail: { count: this.favorites.length, favorites: this.favorites }
    });
    document.dispatchEvent(event);
  }

  onChanged(callback) {
    document.addEventListener('favorites:changed', callback);
    return () => document.removeEventListener('favorites:changed', callback);
  }
}

const favorites = new FavoritesManager();

// Favorite button on movie card
function createMovieCard(movie) {
  const card = document.createElement('div');
  card.className = 'movie-card glass-card';
  card.dataset.id = movie.id;
  const isFav = favorites.isFavorite(movie.id);
  card.innerHTML = `
    

      
      

        ${TMDB.getStarRating(movie.voteAverage)} ${movie.voteAverage.toFixed(1)}
      

      
    

    

      
${movie.title}

      

        ${movie.releaseYear}
        
          ${movie.voteAverage.toFixed(1)}
        
      

    

  `;
  card.querySelector('.movie-poster').addEventListener('click', () => showMovieDetail(movie.id));
  card.querySelector('.fav-btn').addEventListener('click', (e) => {
    e.stopPropagation();
    const btn = e.currentTarget;
    const icon = btn.querySelector('i');
    if (favorites.isFavorite(movie.id)) {
      favorites.remove(movie.id);
      btn.classList.remove('active');
      icon.className = 'far fa-heart';
    } else {
      favorites.add(movie);
      btn.classList.add('active');
      icon.className = 'fas fa-heart';
    }
  });
  return card;
}

Step 5: Trailer Modal with YouTube IFrame

Build a modal system for displaying movie trailers from the TMDB videos endpoint, embedded via the YouTube IFrame Player API.

class TrailerModal {
  constructor() {
    this.modal = document.getElementById('trailer-modal');
    this.overlay = document.getElementById('modal-overlay');
    this.playerContainer = document.getElementById('youtube-player');
    this.player = null;
    this.bindEvents();
  }

  bindEvents() {
    this.overlay?.addEventListener('click', () => this.close());
    document.addEventListener('keydown', (e) => {
      if (e.key === 'Escape') this.close();
    });
  }

  async open(movieId) {
    this.modal?.classList.add('active');
    this.overlay?.classList.add('active');
    document.body.style.overflow = 'hidden';

    if (!this.playerContainer) return;

    try {
      const videos = await TMDB.getMovieVideos(movieId);
      const trailer = videos.results.find(
        v => v.type === 'Trailer' && v.official === true && v.site === 'YouTube'
      ) || videos.results.find(v => v.type === 'Trailer' && v.site === 'YouTube');

      if (trailer) {
        this.playerContainer.innerHTML = '';
        this.player = new YT.Player('youtube-player', {
          height: '100%',
          width: '100%',
          videoId: trailer.key,
          playerVars: {
            autoplay: 1,
            modestbranding: 1,
            rel: 0,
            showinfo: 0
          },
          events: {
            onReady: (event) => event.target.playVideo(),
            onError: () => this.showFallback()
          }
        });
      } else {
        this.showFallback();
      }
    } catch {
      this.showFallback();
    }
  }

  showFallback() {
    this.playerContainer.innerHTML = `
      

        
        
No trailer available for this movie

      

    `;
  }

  close() {
    this.modal?.classList.remove('active');
    this.overlay?.classList.remove('active');
    document.body.style.overflow = '';
    if (this.player) {
      this.player.stopVideo();
      this.player.destroy();
      this.player = null;
    }
    if (this.playerContainer) this.playerContainer.innerHTML = '';
  }
}

// Load YouTube IFrame API asynchronously
function loadYouTubeAPI() {
  return new Promise((resolve) => {
    if (window.YT) { resolve(); return; }
    const tag = document.createElement('script');
    tag.src = 'https://www.youtube.com/iframe_api';
    const firstScript = document.getElementsByTagName('script')[0];
    firstScript.parentNode.insertBefore(tag, firstScript);
    window.onYouTubeIframeAPIReady = resolve;
  });
}

Common Mistakes

1. Race Conditions with Search Results
When the user types quickly, multiple API requests are fired. Without AbortController, a slow response from an earlier query can arrive after a faster response from a later query, causing incorrect results to display. Always cancel the previous request before making a new one. Use AbortController and check if the request was aborted before updating the UI. Additionally, track a request counter and ignore responses that don't match the latest query.
2. Memory Leaks from Infinite Scroll
Loading unlimited pages creates thousands of DOM elements, causing memory issues and slow scrolling. Implement a virtual scrolling technique (react-window) or limit the total number of loaded pages (e.g., max 50 pages). Remove off-screen elements from the DOM when they are far from the viewport. Use DOM node pooling to reuse card elements instead of creating new ones.
3. Broken Images from Missing TMDB Paths
TMDB returns null for poster_path and backdrop_path for some movies (especially lesser-known titles). Without fallback handling, users see broken image icons. Always check for null paths and provide a placeholder image. Use the onerror event handler on img elements to replace broken images with a placeholder. Consider showing a gradient fallback with the movie title as an alternative.
4. YouTube IFrame API Loading Issues
The YouTube IFrame API loads asynchronously and may not be ready when the user clicks "Watch Trailer". Always wait for onYouTubeIframeAPIReady before creating players. Handle cases where YouTube is blocked by corporate firewalls or ad-blockers. Provide a non-YouTube fallback (e.g., just show a "Watch on YouTube" link with the video URL). Destroy the player when the modal closes to free memory.

Interview Questions

Q: How does debouncing differ from throttling? When would you use each?
A: Debouncing delays execution until after a specified quiet period - if events keep firing, the timer resets. Use for search inputs (wait until user stops typing), window resize handlers, and auto-save. Throttling limits execution to at most once per specified interval - events during the cooldown are ignored. Use for scroll handlers, resize events where you need periodic updates, and button click prevention. Key difference: debounce fires once at the end of a burst, throttle fires periodically during the burst.
Q: Explain the Intersection Observer API and its advantages over scroll event listeners.
A: IntersectionObserver observes when a target element intersects with a root element (usually the viewport). Advantages: runs off the main thread (no scroll jank), doesn't fire on every scroll pixel (configurable threshold), works with iframes cross-origin, and has built-in rootMargin for triggering before the element is visible. Use for lazy loading images, infinite scroll, animation triggers, and tracking ad/viewport visibility. Performance: IntersectionObserver uses 0-2% CPU vs scroll listeners which fire at 60fps.
Q: How would you implement an autocomplete search with movie suggestions?
A: Use a debounced search that queries the TMDB search endpoint. Display suggestions in a dropdown positioned below the input. Handle keyboard navigation (arrow keys, Enter, Escape). Click outside the dropdown to close. Use AbortController to cancel stale requests. Highlight matching text in suggestions using substring matching. Cache recent suggestions in a Map with TTL. For better UX, show movie posters alongside titles in suggestions. Limit to 5-8 suggestions with a "View all results" option at the bottom.
Q: What strategies can you use to optimize image loading for a grid with hundreds of movie posters?
A: (1) Lazy loading: loading="lazy" attribute + Intersection Observer to load images when they enter the viewport. (2) Responsive images: use TMDB's size variants (w185, w342, w500) based on card size using srcset. (3) Blur placeholder: show a tiny blurred version (20px wide) as a base64 data URL while the full image loads. (4) Progressive JPEGs load in increasing quality passes. (5) CDN caching: TMDB images are served from a CDN with proper Cache-Control headers. (6) Preload the first visible images with fetchpriority="high" or preload links. (7) Use WebP format if the browser supports it (TMDB returns WebP automatically).
Q: How would you implement offline support for the movie search app?
A: Use a service worker with the Cache API. Cache API responses in the service worker fetch event using a cache-first strategy for images and a network-first strategy for search results (falling back to cached results). Store recently viewed movie details in IndexedDB for offline access. Cache the app shell (HTML, CSS, JS) during the install event. Show a banner when offline and disable search (or search cached results). For favorites, localStorage already works offline. Add a manifest.json for PWA installability.

Extension Ideas

  • Streaming Availability: Integrate the TMDB watch providers endpoint to show where each movie is available to stream (Netflix, Hulu, Prime, Disney+). Add filter by streaming service. Display pricing and subscription requirements.
  • Personalized Recommendations: Implement a recommendation engine based on favorite movies. Find similar movies using TMDB's /movie/{id}/recommendations endpoint. Build a "Because you liked X" section on the home page using collaborative filtering with localStorage favorites.
  • Social Features: Add user reviews and ratings (independent of TMDB). Implement shareable movie lists (public URLs with query params). Add a "Friends' Recommendations" feature using a simple social graph stored in IndexedDB. Integrate with Twitter/WhatsApp sharing for individual movies.
  • Advanced Search Filters: Add search filters for genre (multi-select), year range (slider), minimum rating, language, and adult content toggle. Implement URL-based filter state for shareable filtered searches. Add sort options: popularity, rating, release date, title.
