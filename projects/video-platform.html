Video Streaming Platform

Advanced 60-80 hours 10 technologies Capstone

Project Overview

Build a full-stack video streaming platform similar to YouTube or Vimeo, supporting video upload, transcoding, adaptive bitrate streaming (HLS/DASH), CDN delivery, user management, subscriptions, analytics, and live streaming. The platform uses Node.js/Python for the backend, React for the frontend, FFmpeg for video processing, AWS S3 + CloudFront for storage and delivery, and Redis + PostgreSQL for state and metadata. You will implement a complete video encoding pipeline, adaptive bitrate playlist generation, thumbnail extraction, content delivery optimization via CDN, and a user-facing video player with custom controls. The platform supports 4K uploads, automatic quality switching based on network conditions, and detailed watch-time analytics for content creators.

Problem Statement

Modern video streaming at scale presents interconnected challenges: (1) Video files are large (1GB+ for 4K content) requiring efficient upload with resumable support, (2) Raw video must be transcoded into multiple formats/resolutions for adaptive streaming, (3) Global delivery with sub-second startup latency requires CDN optimization and edge caching, (4) Users on different devices and network conditions need adaptive quality switching, (5) Content creators need analytics — watch time, audience retention, geographic distribution, (6) Monetization requires subscription management, ad insertion, and pay-per-view access control. The goal is to build a platform that handles the entire video lifecycle — from upload to global delivery — with professional-grade quality, security, and analytics.

Real World Use Cases

E-Learning Platform

Course video hosting with DRM protection, chapter markers, interactive quizzes overlayed on video, and watch-time tracking for completion certificates. Used by online course creators.

Live Game Streaming

Real-time game streaming with sub-3-second latency via WebRTC + HLS hybrid. Chat integration, subscriber emotes, and VOD archiving of past streams. Twitch-like experience.

Enterprise Video Training

Internal corporate training videos with SSO integration, SCORM compliance, detailed employee viewing analytics, and mandatory video completion tracking for compliance training.

Video-on-Demand Service

Netflix-like streaming service with content library management, multi-language subtitles, parental controls, recommendation engine, and offline download support for mobile viewing.

Industry Relevance

The global video streaming market is valued at $500B+ with 3.5B+ internet users streaming video regularly. YouTube serves 500+ hours of video uploaded every minute. Netflix accounts for 15% of global internet traffic. Key trends: (1) Live streaming growth — 150% increase since 2020, (2) Short-form video (TikTok, Reels, Shorts) driving new encoding and delivery paradigms, (3) AI-powered video — automatic highlight generation, content moderation, and recommendation, (4) Serverless video processing — AWS Elemental MediaConvert, Lambda-based transcoding, (5) Low-latency streaming — WebRTC and LL-HLS reducing latency to sub-2 seconds. This project builds skills for video infrastructure roles at YouTube, Twitch, Netflix, and media companies.

Step 1: Multipart Upload with Presigned URLs

// upload-service/src/routes/upload.ts import { Router } from 'express'; import { S3Client, CreateMultipartUploadCommand, UploadPartCommand, CompleteMultipartUploadCommand, AbortMultipartUploadCommand } from '@aws-sdk/client-s3'; import { getSignedUrl } from '@aws-sdk/s3-request-presigner'; const router = Router(); const s3 = new S3Client({ region: process.env.AWS_REGION }); const BUCKET = process.env.RAW_VIDEO_BUCKET!; // Initiate multipart upload router.post('/upload/initiate', async (req, res) => { const { fileName, fileSize, mimeType } = req.body; const userId = req.user.id; // Create S3 multipart upload const command = new CreateMultipartUploadCommand({ Bucket: BUCKET, Key: `uploads/${userId}/${Date.now()}-${fileName}`, ContentType: mimeType, Metadata: { userId, originalName: fileName, fileSize: String(fileSize) }, }); const { UploadId, Key } = await s3.send(command); // Generate presigned URLs for each part const partCount = Math.ceil(fileSize / (5 * 1024 * 1024)); // 5MB parts const presignedUrls = await Promise.all( Array.from({ length: partCount }, (_, i) => getSignedUrl(s3, new UploadPartCommand({ Bucket: BUCKET, Key, UploadId, PartNumber: i + 1, }), { expiresIn: 3600 }) ) ); res.json({ uploadId: UploadId, key: Key, presignedUrls, partCount, }); }); // Complete multipart upload router.post('/upload/complete', async (req, res) => { const { key, uploadId, parts } = req.body; const command = new CompleteMultipartUploadCommand({ Bucket: BUCKET, Key: key, UploadId: uploadId, MultipartUpload: { Parts: parts.map((p: any) => ({ ETag: p.etag, PartNumber: p.partNumber })) }, }); const result = await s3.send(command); // Trigger transcoding pipeline await fetch(`${process.env.API_URL}/api/videos/process`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ key: result.Key, bucket: BUCKET }), }); res.json({ location: result.Location, key: result.Key }); }); // Abort multipart upload router.post('/upload/abort', async (req, res) => { const { key, uploadId } = req.body; await s3.send(new AbortMultipartUploadCommand({ Bucket: BUCKET, Key: key, UploadId: uploadId, })); res.json({ status: 'aborted' }); }); export default router;

await new Promise((resolve, reject) => { const ffmpeg = spawn('ffmpeg', ffmpegArgs, { stdio: ['pipe', 'pipe', 'pipe'] }); // Pipe S3 stream to ffmpeg pipeline(inputStream, ffmpeg.stdin!).catch(reject); resolve(); } else { reject(new Error(`FFmpeg exited with code ${code}: ${stderr}`)); } }); }); } // Generate master playlist const masterPlaylist = [ '#EXTM3U', '#EXT-X-VERSION:3', ...variantPlaylists, ].join('\n'); const masterKey = `${config.outputPrefix}/master.m3u8`; await s3.send(new PutObjectCommand({ Bucket: config.bucket, Key: masterKey, Body: masterPlaylist, ContentType: 'application/vnd.apple.mpegurl', })); return masterKey; }

Video Encoding Pipelines

FFmpeg Pipeline: Raw video ? (1) Detect codec and container, (2) Generate keyframe index for seeking, (3) Transcode to H.264/AAC for maximum compatibility, (4) Generate multiple renditions at 1080p/720p/480p/360p, (5) Segment into 6-second HLS segments, (6) Generate master playlist and variant playlists, (7) Extract thumbnail at 25%, 50%, 75% positions, (8) Generate sprite sheet for timeline preview. The pipeline runs on ECS Fargate with GPU support for faster encoding. For 4K content, we also generate a 4K rendition using HEVC (H.265) codec for 50% bitrate savings.

CDN Concepts (CloudFront, Cloudflare)

CloudFront Distribution: (1) Origin: S3 bucket with HLS content, (2) Behavior: /hls/* ? Cached for 1 year (versioned content), /thumbnails/* ? Cached 1 day, /api/* ? Proxy to API Gateway, (3) Origin Shield: Reduces S3 request cost by consolidating requests at regional edge, (4) Price Class: Use Price Class 100 (US, Europe) for cost savings, (5) Lambda@Edge: For token authorization on premium content, (6) WAF: Block malicious requests, geo-restriction for licensed content.

Video Storage Strategies (S3, GCS)

Tiered Storage: (1) Hot tier (S3 Standard): Videos published within 30 days, popular content. (2) Warm tier (S3 Infrequent Access): Videos 30-90 days old, moderate access. (3) Cold tier (S3 Glacier Deep Archive): Videos older than 90 days, archival. Lifecycle policies automatically transition objects. For originals (raw uploads), delete after successful transcoding or move to Glacier immediately.

Streaming Protocols (HLS, RTMP, MPEG-DASH)

HLS (HTTP Live Streaming): Apple's protocol, widest support. Uses .ts segments with .m3u8 playlists. Supports adaptive bitrate, captions, and DRM (FairPlay). Our primary delivery format.

MPEG-DASH: International standard (ISO/IEC 23009-1). Uses .m4s segments with .mpd manifest. Better codec support (VP9, AV1) and more flexible adaptation sets. Used for Android and smart TV apps.

RTMP (Real-Time Messaging Protocol): Legacy protocol for live streaming ingest. We receive RTMP from OBS/Streamlabs, then repackage to HLS for delivery. Being replaced by SRT (Secure Reliable Transport) and WHIP (WebRTC HTTP Ingest Protocol).

Adaptive Bitrate Streaming

ABR algorithms dynamically select the best quality based on network conditions. Our implementation (using HLS.js built-in ABR): (1) Measures download bandwidth using xhr progress events, (2) Uses EWMA (Exponentially Weighted Moving Average) to smooth bandwidth estimates, (3) Selects highest quality that can be sustained with 20% headroom, (4) On buffer depletion, immediately switch down, (5) On stable high bandwidth for 10 seconds, switch up, (6) Configurable: start with lower quality for fast startup (startLevel = -1 = auto), (7) User can override to fixed quality via quality selector.

Transcoding Workflows

Workflow: (1) S3 event notification triggers SQS message when raw upload completes. (2) Transcode worker picks up message, downloads video metadata (duration, codec, resolution). (3) Determines optimal encoding ladder based on source resolution and bitrate. (4) Runs FFmpeg pipeline — outputs HLS segments to temporary storage. (5) Uploads segments to S3 HLS bucket with appropriate cache headers. (6) Generates thumbnail strip (10 screenshots evenly spaced). (7) Updates video status in PostgreSQL to 'ready'. (8) Invalidates CloudFront cache for the video's CDN path. (9) Optionally deletes raw upload.

Analytics Pipeline

Watch events are sent from the video player to a Kafka topic every 10 seconds (heartbeat). Events include: play, pause, seek, quality_change, heartbeat, complete. Kafka consumer batch-inserts into ClickHouse (columnar analytics DB) every second. Daily aggregation jobs produce per-video and per-creator analytics. Dashboards show: (1) Views over time, (2) Audience retention (watch percentage per timestamp), (3) Geographic distribution (CloudFront logs), (4) Device breakdown, (5) Quality distribution, (6) Revenue per video, (7) Subscriber growth.

Subscription/Billing Integration

Stripe integration: (1) Creator subscription tiers: Basic ($4.99/mo), Premium ($9.99/mo), Enterprise ($19.99/mo). (2) Webhook handling for subscription lifecycle (created, updated, canceled, past_due). (3) Per-viewer entitlements checked via middleware on video access. (4) Revenue sharing: 70% to creator, 30% platform fee. (5) Payouts via Stripe Connect to creator bank accounts. (6) Free tier: ads before video (VAST/VPAID ad server integration).

Security Considerations

  • Signed URLs: All video content served via CloudFront signed URLs with expiration (24h for regular viewing, 7d for offline download). Tokens are tied to user sessions.
  • DRM: Widevine (Android/Chrome), FairPlay (iOS/Safari), and PlayReady (Xbox/Edge) integration using AWS MediaConvert for encrypting content.
  • Upload Validation: Validate file type, scan for malware (ClamAV Lambda), check duration limits (free: 15min, premium: 4h), size limits (free: 1GB, premium: 10GB).
  • Content Moderation: Automated moderation using AWS Rekognition for NSFW content. Manual review queue for flagged videos. Age restriction for mature content.

Common Mistakes

1. Using a single bitrate: Without adaptive streaming, viewers on slow connections buffer endlessly while fast connections get lower quality than possible. Always generate multiple renditions (at least 3: 480p, 720p, 1080p).
2. Not optimizing for mobile: Mobile networks have higher latency and variable bandwidth. Use segment duration of 4s for live (lower latency) and 6s for VOD (better compression). Implement mobile-specific ABR parameters.
3. Ignoring CDN cache hit ratio: Without proper cache headers, every request goes to origin. Set Cache-Control: max-age=31536000 for HLS segments (they're immutable). Use origin shield to reduce S3 costs.
4. No failover for live streams: Live streaming must handle encoder disconnection, network issues, and server failure. Implement auto-reconnect with sequence number continuity. Use redundant ingest endpoints (primary + backup RTMP URLs).
5. Storing all renditions permanently: Storage costs for multiple renditions add up fast. Delete low-bitrate renditions for old/unpopular content after 30 days. Regenerate on-demand if needed.

Advanced Challenges

Challenge 1: Serverless Transcoding Pipeline Replace ECS-based transcoding with AWS Lambda + MediaConvert. Process videos up to 1 hour in Lambda (use EFS for temporary storage, or stream directly). Compare cost and performance vs EC2-based transcoding.
Challenge 2: Real-Time Content Moderation Build a moderation pipeline using FFmpeg scene detection + AWS Rekognition. Automatically flag content containing: weapons, violence, adult content, or copyrighted material. Implement human review queue with justification requirement for takedowns.
Challenge 3: Peer-Assisted Delivery (P2P CDN) Implement WebRTC-based peer-to-peer video delivery to reduce CDN costs. Use WebTorrent or custom WebRTC mesh for popular content. Serve 30% of traffic via peers. Handle peer churn and NAT traversal.
Challenge 4: AI-Powered Video Search Implement semantic video search using CLIP embeddings. Generate embeddings for each video frame at key intervals. Allow natural language search: "show me when the presenter talks about databases". Build a search interface with timestamp results and preview thumbnails.
Challenge 5: Multi-CDN Load Balancing Route traffic across CloudFront, Cloudflare, and Fastly based on real-time performance metrics. Implement client-side CDN selector using JavaScript that measures latency to each CDN's probe URL. Failover to backup CDN within 5 seconds.
