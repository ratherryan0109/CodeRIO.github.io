MySQL

MySQL is the world's most popular open-source relational database management system. Developed originally by MySQL AB (founded by David Axmark, Allan Larsson, and Michael Widenius) and now owned by Oracle Corporation, MySQL powers millions of web applications, from personal blogs to large-scale enterprise systems.

Overview

MySQL was first released in 1995 and has grown to become the de facto standard database for web applications, particularly in the LAMP (Linux, Apache, MySQL, PHP/Perl/Python) stack. It is known for its reliability, ease of use, and strong performance for read-heavy workloads. MySQL supports multiple storage engines, with InnoDB being the default since MySQL 5.5, providing ACID-compliant transactions and foreign key support.

MySQL uses a client-server architecture where the MySQL server manages databases, tables, and handles client connections. It supports standard SQL with extensions like LIMIT, INSERT ... ON DUPLICATE KEY UPDATE, and various storage-engine-specific features. MySQL excels in replication (master-slave, group replication), partitioning, and full-text search capabilities.

Major platforms built on MySQL include Facebook (for some of its core infrastructure), Twitter (originally), YouTube, WordPress (over 43% of the web), Drupal, Joomla, and Magento. MySQL's ecosystem includes tools like phpMyAdmin, MySQL Workbench for GUI management, Percona Toolkit for optimization, and proxies like ProxySQL for load balancing.

Notes

1. MySQL Storage Engines

MySQL supports multiple storage engines. InnoDB is the default and recommended engine for most workloads.

-- Check available engines
SHOW ENGINES;

-- Specify engine at table creation
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL
) ENGINE=InnoDB;

-- InnoDB: ACID, transactions, foreign keys, row-level locking
-- MyISAM: Full-text search, table-level locking, no transactions
-- MEMORY: Data in RAM, fast but non-persistent

2. MySQL-Specific Syntax Extensions

MySQL extends standard SQL with convenient syntax like INSERT ... ON DUPLICATE KEY UPDATE and REPLACE.

-- INSERT or UPDATE if duplicate key
INSERT INTO users (id, name, email)
VALUES (1, 'Alice', 'alice@example.com')
ON DUPLICATE KEY UPDATE
    name = VALUES(name),
    email = VALUES(email);

-- REPLACE (deletes and re-inserts)
REPLACE INTO users (id, name, email)
VALUES (1, 'Bob', 'bob@example.com');

-- INSERT IGNORE (silently skip duplicates)
INSERT IGNORE INTO users (id, name) VALUES (1, 'Charlie');

3. Full-Text Search

MySQL supports full-text indexes for efficient text search on InnoDB and MyISAM tables.

-- Create full-text index
CREATE TABLE articles (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200),
    body TEXT,
    FULLTEXT index ft_search (title, body)
) ENGINE=InnoDB;

-- Natural language mode (default)
SELECT * FROM articles
WHERE MATCH(title, body) AGAINST('database optimization');

-- Boolean mode (operators: +, -, *, "")
SELECT * FROM articles
WHERE MATCH(title, body)
AGAINST('+mysql -oracle' IN BOOLEAN MODE);

4. EXPLAIN Query Analysis

Use EXPLAIN to understand how MySQL executes a query and identify performance bottlenecks.

EXPLAIN SELECT u.name, COUNT(o.id) AS orders
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.created_at > '2024-01-01'
GROUP BY u.id
ORDER BY orders DESC
LIMIT 10;

-- Key columns in output:
-- type: ALL (full scan) | range | ref | eq_ref | const
-- key: which index was used
-- rows: estimated rows examined
-- Extra: Using where, Using index, Using filesort

5. Window Functions in MySQL 8.0+

MySQL 8.0 introduced window functions for analytics without collapsing rows.

SELECT
    name,
    department,
    salary,
    ROW_NUMBER() OVER (PARTITION BY department
                       ORDER BY salary DESC) AS rank_in_dept,
    FIRST_VALUE(salary) OVER (PARTITION BY department
                              ORDER BY salary DESC) AS top_salary,
    LAG(salary, 1) OVER (PARTITION BY department
                         ORDER BY salary) AS prev_salary,
    AVG(salary) OVER (PARTITION BY department) AS dept_avg
FROM employees;

6. JSON Data Type

MySQL's JSON type allows storing and querying JSON documents with specialized functions.

CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    attributes JSON
);

INSERT INTO products VALUES
(1, 'Laptop', '{"color": "silver", "ram": 16, "ports": ["USB-C", "HDMI"]}'),
(2, 'Mouse', '{"color": "black", "wireless": true}');

-- JSON path queries
SELECT name,
       attributes->>'$.color' AS color,
       attributes->>'$.ram' AS ram
FROM products
WHERE JSON_EXTRACT(attributes, '$.wireless') = true;

7. CTEs and Recursive Queries

MySQL 8.0+ supports common table expressions, including recursive CTEs for hierarchical data.

WITH RECURSIVE org_chart AS (
    -- Anchor: top-level employees
    SELECT id, name, manager_id, 1 AS level
    FROM employees
    WHERE manager_id IS NULL

    UNION ALL

    -- Recursive: employees with managers
    SELECT e.id, e.name, e.manager_id, oc.level + 1
    FROM employees e
    JOIN org_chart oc ON e.manager_id = oc.id
)
SELECT * FROM org_chart ORDER BY level, name;

8. Performance Schema

The performance_schema provides runtime metrics for monitoring MySQL's internal operations.

-- Top queries by execution time
SELECT DIGEST_TEXT, COUNT_STAR, SUM_TIMER_WAIT/1000000000 AS total_ms
FROM performance_schema.events_statements_summary_by_digest
ORDER BY SUM_TIMER_WAIT DESC
LIMIT 10;

-- Table I/O statistics
SELECT OBJECT_SCHEMA, OBJECT_NAME,
       COUNT_READ, COUNT_WRITE,
       SUM_TIMER_READ/1000000000 AS read_ms
FROM performance_schema.table_io_waits_summary_by_table
ORDER BY SUM_TIMER_WAIT DESC
LIMIT 10;

Examples

Example 1: E-Commerce Schema & Queries

A complete MySQL schema for an e-commerce platform with relevant queries.

CREATE TABLE customers (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE orders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT NOT NULL,
    total DECIMAL(10,2) NOT NULL,
    status ENUM('pending','shipped','delivered') DEFAULT 'pending',
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (customer_id) REFERENCES customers(id)
);

CREATE TABLE order_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id INT NOT NULL,
    product_name VARCHAR(200),
    quantity INT NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(id)
);

-- Monthly revenue report
SELECT DATE_FORMAT(order_date, '%Y-%m') AS month,
       COUNT(DISTINCT o.id) AS orders,
       SUM(oi.quantity * oi.price) AS revenue
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
WHERE o.status != 'cancelled'
GROUP BY month
ORDER BY month DESC;

Example 2: Stored Procedure

A stored procedure for paginated customer search with total count.

DELIMITER $$

CREATE PROCEDURE search_customers(
    IN search_term VARCHAR(100),
    IN page_num INT,
    IN page_size INT
)
BEGIN
    DECLARE offset_val INT;
    SET offset_val = (page_num - 1) * page_size;

    -- Total count
    SELECT COUNT(*) AS total_records
    FROM customers
    WHERE name LIKE CONCAT('%', search_term, '%')
       OR email LIKE CONCAT('%', search_term, '%');

    -- Paginated results
    SELECT id, name, email, created_at
    FROM customers
    WHERE name LIKE CONCAT('%', search_term, '%')
       OR email LIKE CONCAT('%', search_term, '%')
    ORDER BY name
    LIMIT page_size OFFSET offset_val;
END$$

DELIMITER ;

CALL search_customers('john', 1, 20);
