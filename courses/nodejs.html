Node.js

Node.js is an open-source, cross-platform JavaScript runtime built on Chrome's V8 JavaScript engine. Created by Ryan Dahl in 2009, Node.js revolutionized server-side programming by bringing JavaScript to the backend. Its event-driven, non-blocking I/O model makes it ideal for building scalable network applications, real-time services, and microservices.

Overview

Node.js uses an event-driven, non-blocking I/O architecture that makes it efficient for handling many concurrent connections without the overhead of thread-based servers. Its core philosophy is that I/O operations (database, filesystem, network) are the bottleneck, so Node.js delegates them to background tasks and processes the callback when complete, never blocking the main event loop.

The Node.js ecosystem includes npm (Node Package Manager), the largest package registry in the world with over 2 million packages. Frameworks like Express.js (minimalist), NestJS (structured, TypeScript-first), Fastify (fast), and Socket.IO (real-time) provide building blocks for different application types. Node.js also supports TypeScript natively through ts-node or compilation.

Node.js powers critical infrastructure at Netflix (streaming optimization), LinkedIn (mobile backend), PayPal (payment processing), Uber (trip management), Walmart (e-commerce), and Medium (content delivery). Its versatility extends to serverless computing (AWS Lambda, Vercel), desktop applications (Electron), and command-line tools. The Node.js event loop, libuv library, and worker threads (added in Node 10.5) provide the concurrency model.

1. The Node.js Event Loop

The event loop is the core of Node.js async model. It processes phases in order: timers, pending callbacks, idle/prepare, poll, check, close callbacks.

const fs = require('fs');
const start = Date.now();

setTimeout(() => console.log('timer 1'), 0);
setImmediate(() => console.log('immediate 1'));

fs.readFile(__filename, () => {
    console.log('I/O callback');
    setTimeout(() => console.log('timer from I/O'), 0);
    setImmediate(() => console.log('immediate from I/O'));
});

console.log('main thread');
// Order: main thread, timer 1, immediate 1, I/O callback,
//        immediate from I/O, timer from I/O

2. Express Router & Middleware

Express middleware functions execute in order. Routes can be modularized using Router.

const express = require('express');
const router = express.Router();

// Middleware - runs for all routes on this router
router.use((req, res, next) => {
    console.log(`${req.method} ${req.url}`);
    next();
});

// Route handlers
router.get('/users', async (req, res, next) => {
    try {
        const users = await db.findMany();
        res.json(users);
    } catch (err) {
        next(err); // Pass to error handler
    }
});

// Error handling middleware
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).json({ error: err.message });
});

3. Async/Await Error Handling

Proper error handling in async code requires try/catch blocks or a wrapper function to catch rejected promises.

// Wrapper for async route handlers
const asyncHandler = (fn) => (req, res, next) => {
    Promise.resolve(fn(req, res, next)).catch(next);
};

app.get('/users/:id', asyncHandler(async (req, res) => {
    const user = await User.findById(req.params.id);
    if (!user) {
        return res.status(404).json({ error: 'Not found' });
    }
    res.json(user);
}));

// Alternative: try/catch in every handler
app.get('/posts', async (req, res, next) => {
    try {
        const posts = await Post.find().sort('-createdAt');
        res.json(posts);
    } catch (err) {
        next(err);
    }
});

4. Streams for Large Data

Streams process data piece by piece, avoiding loading entire files into memory. Perfect for large files or network responses.

const fs = require('fs');
const zlib = require('zlib');
const { Transform } = require('stream');

// Chain streams: read -> transform -> compress -> write
fs.createReadStream('input.txt')
    .pipe(new Transform({
        transform(chunk, encoding, callback) {
            this.push(chunk.toString().toUpperCase());
            callback();
        }
    }))
    .pipe(zlib.createGzip())
    .pipe(fs.createWriteStream('output.txt.gz'))
    .on('finish', () => console.log('Done'));
    .on('error', (err) => console.error(err));

5. JWT Authentication

JSON Web Tokens provide stateless authentication. The server signs a token that the client sends with each request.

const jwt = require('jsonwebtoken');
const SECRET = process.env.JWT_SECRET;

// Sign a token
function generateToken(user) {
    return jwt.sign(
        { id: user.id, role: user.role },
        SECRET,
        { expiresIn: '7d' }
    );
}

// Verify middleware
function authenticate(req, res, next) {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ error: 'No token' });

    try {
        req.user = jwt.verify(token, SECRET);
        next();
    } catch (err) {
        res.status(401).json({ error: 'Invalid token' });
    }
}

7. Socket.IO Real-Time

Socket.IO enables real-time bidirectional communication between clients and servers with fallback to HTTP long-polling.

const io = require('socket.io')(server, {
    cors: { origin: '*' }
});

io.on('connection', (socket) => {
    console.log('User connected:', socket.id);

    socket.on('join-room', (room) => {
        socket.join(room);
        socket.to(room).emit('user-joined', socket.id);
    });

    socket.on('send-message', (data) => {
        // Broadcast to room except sender
        socket.to(data.room).emit('new-message', {
            user: socket.id,
            text: data.text,
            time: Date.now()
        });
    });

    socket.on('disconnect', () => {
        console.log('User disconnected:', socket.id);
    });
});

8. Worker Threads

Worker threads run CPU-intensive JavaScript in parallel, offloading work from the main event loop.

// worker.js
const { parentPort } = require('worker_threads');

parentPort.on('message', (n) => {
    const result = fibonacci(n);
    parentPort.postMessage(result);
});

function fibonacci(n) {
    return n < 2 ? n : fibonacci(n - 1) + fibonacci(n - 2);
}

// main.js
const { Worker } = require('worker_threads');

function runWorker(n) {
    return new Promise((resolve, reject) => {
        const worker = new Worker('./worker.js');
        worker.postMessage(n);
        worker.on('message', resolve);
        worker.on('error', reject);
    });
}

const result = await runWorker(40);
console.log('Fibonacci:', result);

Examples

Example 1: Express REST API

A complete CRUD API using Express and in-memory storage.

const express = require('express');
const app = express();
app.use(express.json());

let todos = [];
let id = 1;

app.get('/api/todos', (req, res) => {
    const { completed } = req.query;
    const filtered = completed !== undefined
        ? todos.filter(t => t.completed === (completed === 'true'))
        : todos;
    res.json(filtered);
});

app.post('/api/todos', (req, res) => {
    const { title } = req.body;
    if (!title) return res.status(400).json({ error: 'Title required' });
    const todo = { id: id++, title, completed: false };
    todos.push(todo);
    res.status(201).json(todo);
});

app.put('/api/todos/:id', (req, res) => {
    const todo = todos.find(t => t.id === parseInt(req.params.id));
    if (!todo) return res.status(404).json({ error: 'Not found' });
    todo.completed = req.body.completed ?? todo.completed;
    todo.title = req.body.title ?? todo.title;
    res.json(todo);
});

app.delete('/api/todos/:id', (req, res) => {
    const idx = todos.findIndex(t => t.id === parseInt(req.params.id));
    if (idx === -1) return res.status(404).json({ error: 'Not found' });
    todos.splice(idx, 1);
    res.status(204).send();
});

app.listen(3000, () => console.log('Server on :3000'));

Example 2: CLI Tool with Commander

A command-line file search tool using the commander library.

#!/usr/bin/env node
const { program } = require('commander');
const fs = require('fs/promises');
const path = require('path');

program
    .name('file-search')
    .description('Search files by pattern')
    .argument('', 'search pattern')
    .option('-d, --dir ', 'directory to search', '.')
    .option('-r, --recursive', 'search recursively')
    .action(async (pattern, options) => {
        const files = await searchFiles(options.dir, pattern, options.recursive);
        files.forEach(f => console.log(f));
    });

async function searchFiles(dir, pattern, recursive) {
    const entries = await fs.readdir(dir, { withFileTypes: true });
    const results = [];
    for (const entry of entries) {
        const fullPath = path.join(dir, entry.name);
        if (entry.isFile() && entry.name.includes(pattern)) {
            results.push(fullPath);
        } else if (entry.isDirectory() && recursive) {
            results.push(...await searchFiles(fullPath, pattern, true));
        }
    }
    return results;
}

program.parse();
