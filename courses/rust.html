Rust

Rust is a systems programming language focused on safety, speed, and concurrency. Developed by Mozilla Research and first stabilized in 2015, Rust guarantees memory safety without a garbage collector through its unique ownership and borrowing system. It has been voted the most loved language on Stack Overflow every year since 2016.

Overview

Rust was created by Graydon Hoare at Mozilla Research, with the first stable release (1.0) in May 2015. It was designed to address the fundamental tension between safety and control in systems programming. Rust's key innovation is its ownership system: every value has a single owner, and the compiler enforces rules about how references (borrows) work at compile time, eliminating entire categories of bugs such as use-after-free, double-free, and data races.

Rust compiles to native code via LLVM, delivering performance comparable to C and C++. Its package manager and build tool, Cargo, provides dependency management, testing, documentation generation, and benchmarking out of the box. Rust's pattern matching, algebraic data types (enums with data), and trait-based generics make it expressive enough for high-level abstractions while maintaining zero-cost execution.

Rust is used by Firefox for CSS engine (Servo/Stylo), AWS for infrastructure (Firecracker, Bottlerocket), Dropbox for file sync, Figma for performance-critical rendering, Discord for read-state storage, and Cloudflare for edge computing. The Linux kernel has begun accepting Rust code as a second language for drivers, and Microsoft is using Rust to replace C# components in Azure.

Notes

1. Ownership Rules

Rust's ownership system has three fundamental rules that the compiler enforces at compile time.

fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 is MOVED to s2

    // println!("{}", s1); // ERROR: borrow of moved value
    println!("{}", s2);   // OK

    let s3 = s2.clone();  // deep copy (expensive)
    println!("{} {}", s2, s3); // both valid
}

2. Borrowing and References

References allow you to access data without taking ownership. At any time, you can have either one mutable reference or any number of immutable references.

fn calculate_length(s: &String) -> usize {
    s.len()  // borrows, not owns
}  // s goes out of scope, but nothing happens

fn main() {
    let mut s = String::from("hello");
    let r1 = &s;      // immutable borrow
    let r2 = &s;      // another immutable borrow (OK)
    // let r3 = &mut s; // ERROR: cannot borrow as mutable
    println!("{} {}", r1, r2);
    let r3 = &mut s;  // OK: no immutable borrows in scope
    r3.push_str(" world");
}

3. Pattern Matching with Enums

Rust enums are algebraic data types that can hold data. Match expressions exhaustively handle all variants.

enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(u8, u8, u8),
}

fn handle_msg(msg: Message) {
    match msg {
        Message::Quit => println!("Quitting"),
        Message::Move { x, y } => {
            println!("Move to ({}, {})", x, y);
        }
        Message::Write(text) => println!("{}", text),
        Message::ChangeColor(r, g, b) => {
            println!("Color: {} {} {}", r, g, b);
        }
    }
}

4. Error Handling with Result

Rust encourages explicit error handling through the Result type. The ? operator propagates errors automatically.

use std::fs::File;
use std::io::{self, Read};

fn read_username(path: &str) -> Result {
    let mut file = File::open(path)?; // propagates error
    let mut content = String::new();
    file.read_to_string(&mut content)?;
    Ok(content.trim().to_string())
}

fn main() {
    match read_username("user.txt") {
        Ok(name) => println!("User: {}", name),
        Err(e) => eprintln!("Failed: {}", e),
    }
}

5. Traits and Generics

Traits define shared behavior. Generics with trait bounds allow polymorphic functions without runtime overhead.

trait Summary {
    fn summarize(&self) -> String;
}

struct Article {
    title: String,
    content: String,
}

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{}: {}...", self.title, &self.content[..20])
    }
}

fn notify(item: T) {
    println!("Breaking: {}", item.summarize());
}

6. Lifetimes

Lifetime annotations tell the borrow checker how long references are valid. They ensure references don't outlive the data they point to.

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

fn main() {
    let s1 = String::from("short");
    let result;
    {
        let s2 = String::from("longer");
        result = longest(&s1, &s2);
        println!("Longest: {}", result);
    }
    // println!("{}", result); // ERROR: s2 dropped
}

7. Closures and Iterators

Closures are anonymous functions that capture their environment. Iterators enable lazy, chainable data processing.

let numbers = vec![1, 2, 3, 4, 5, 6];

let even_sum: i32 = numbers
    .iter()
    .filter(|&x| x % 2 == 0)
    .map(|x| x * x)
    .sum();

println!("Sum of even squares: {}", even_sum);

let expensive = |x: i32| -> i32 {
    println!("computing...");
    x * 2
};

8. Concurrency with Channels

Rust's std::sync::mpsc channels enable safe message passing between threads, following Go's CSP model.

use std::sync::mpsc;
use std::thread;

let (tx, rx) = mpsc::channel();

thread::spawn(move || {
    tx.send(String::from("hello")).unwrap();
});

let received = rx.recv().unwrap();
println!("Got: {}", received);

Examples

Example 1: CLI Todo Application

A simple command-line todo list using structs and file I/O.

use std::fs;
use std::io::{self, Write};

struct Todo {
    items: Vec,
}

impl Todo {
    fn new() -> Self {
        Todo { items: Vec::new() }
    }

    fn add(&mut self, item: String) {
        self.items.push(item);
    }

    fn save(&self, path: &str) -> io::Result<()> {
        let content = self.items.join("\n");
        fs::write(path, content)
    }

    fn load(path: &str) -> io::Result {
        let content = fs::read_to_string(path)?;
        let items = content.lines().map(String::from).collect();
        Ok(Todo { items })
    }
}

fn main() -> io::Result<()> {
    let mut todo = Todo::new();
    todo.add("Learn Rust".to_string());
    todo.add("Build a project".to_string());
    todo.save("todo.txt")?;
    println!("Saved {} tasks", todo.items.len());
    Ok(())
}

Example 2: Simple Web Server

A basic HTTP server using the standard library.

use std::net::{TcpListener, TcpStream};
use std::io::{Read, Write};

fn handle_client(mut stream: TcpStream) {
    let mut buffer = [0; 1024];
    stream.read(&mut buffer).unwrap();

    let response = "HTTP/1.1 200 OK\r\nContent-Type: text/html\r\n\r\n
Hello from Rust!
";
    stream.write(response.as_bytes()).unwrap();
    stream.flush().unwrap();
}

fn main() -> io::Result<()> {
    let listener = TcpListener::bind("127.0.0.1:8080")?;
    for stream in listener.incoming() {
        handle_client(stream?);
    }
    Ok(())
}
