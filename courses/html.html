Overview

HTML, short for HyperText Markup Language, is the standard markup language used to create web pages and web applications. Conceived by Tim Berners-Lee in 1991, HTML has evolved through multiple versions — from HTML 2.0 in 1995 to HTML5 in 2014, which introduced semantic elements, native multimedia support, and powerful APIs.

HTML uses a system of tags enclosed in angle brackets to define elements such as headings, paragraphs, links, images, forms, and tables. These elements tell the browser how to structure and display content. Every web page on the internet relies on HTML as its foundational layer, with CSS enhancing presentation and JavaScript adding interactivity.

Modern HTML5 supports rich media embedding via <video> and <audio>, vector graphics with <svg> and <canvas>, offline storage through localStorage and sessionStorage, geolocation APIs, drag-and-drop, web workers, and semantic elements like <header>, <nav>, <main>, <section>, <article>, and <footer> that improve accessibility and SEO.

Notes

1. HTML Document Structure

Every HTML document follows a basic structure. The DOCTYPE declaration must be the first line, followed by the html element containing head and body sections.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Page Title</title>
</head>
<body>
  <!-- Visible content goes here -->
</body>
</html>

2. Semantic vs Non-Semantic Elements

Semantic elements clearly describe their meaning to both the browser and developer. Non-semantic elements like div and span tell nothing about their content.

<!-- Non-semantic -->
<div class="header">...</div>
<div class="nav">...</div>

<!-- Semantic HTML5 -->
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>

3. Forms and Input Types

HTML5 introduced many new input types that enable native validation and better user experience across devices.

<form action="/submit" method="POST">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="age">Age:</label>
  <input type="number" id="age" name="age" min="1" max="120">

  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="us">United States</option>
    <option value="in">India</option>
  </select>

  <button type="submit">Submit</button>
</form>

4. HTML5 Multimedia

Native audio and video elements eliminate the need for Flash plugins. Multiple source formats ensure cross-browser compatibility.

<video controls width="640">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support video.
</video>

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support audio.
</audio>

5. Tables with Structure

HTML tables should use thead, tbody, and tfoot for proper structure and accessibility.

<table>
  <thead>
    <tr><th>Name</th><th>Age</th><th>City</th></tr>
  </thead>
  <tbody>
    <tr><td>Alice</td><td>28</td><td>NYC</td></tr>
    <tr><td>Bob</td><td>34</td><td>London</td></tr>
  </tbody>
  <tfoot>
    <tr><td colspan="3">2 records</td></tr>
  </tfoot>
</table>

6. Data Attributes

Custom data attributes store extra information in HTML elements, accessible via JavaScript.

<article data-id="42" data-author="john" data-published="2024-01-15">
  <h2>Article Title</h2>
  <p>Content here...</p>
</article>

<script>
  const article = document.querySelector('article');
  console.log(article.dataset.id);      // "42"
  console.log(article.dataset.author);  // "john"
</script>

7. Responsive Images

The srcset and sizes attributes allow browsers to choose the optimal image based on viewport and device pixel ratio.

<img src="image-640.jpg"
     srcset="image-320.jpg 320w,
             image-640.jpg 640w,
             image-1280.jpg 1280w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1200px) 50vw,
            33vw"
     alt="Responsive example">

8. HTML Entities & Special Characters

HTML entities represent reserved characters and special symbols. They start with & and end with ;

&lt;  = <       &gt;  = >
&amp; = &       &quot; = "
&copy; = ©   &reg; = ®
&nbsp; = non-breaking space
&#10003; = ✓    &#9733; = ★

Examples

Example 1: Personal Portfolio Page

A complete semantic HTML portfolio page structure.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>John Doe - Portfolio</title>
</head>
<body>
  <header>
    <nav>
      <ul>
        <li><a href="#about">About</a></li>
        <li><a href="#projects">Projects</a></li>
        <li><a href="#contact">Contact</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section id="about">
      <h1>Hi, I'm John</h1>
      <p>A web developer passionate about building accessible websites.</p>
    </section>

    <section id="projects">
      <h2>Projects</h2>
      <article>
        <h3>Project One</h3>
        <p>Description of the project.</p>
      </article>
    </section>

    <section id="contact">
      <h2>Contact Me</h2>
      <form action="/submit" method="POST">
        <label for="name">Name:</label>
        <input type="text" id="name" name="name" required>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
        <button type="submit">Send</button>
      </form>
    </section>
  </main>

  <footer>
    <p>&copy; 2025 John Doe</p>
  </footer>
</body>
</html>

Example 2: Product Card with Data Attributes

A product card using semantic elements and data attributes for JavaScript interaction.

<article class="product-card" data-id="101" data-category="electronics" data-price="29.99">
  <figure>
    <img src="product.jpg" alt="Wireless Mouse">
    <figcaption>Ergonomic Wireless Mouse</figcaption>
  </figure>
  <div class="product-info">
    <h3>Wireless Mouse Pro</h3>
    <p class="price">$29.99</p>
    <ul class="features">
      <li>Bluetooth 5.0</li>
      <li>6-month battery</li>
      <li>Silent clicks</li>
    </ul>
    <button class="add-to-cart" data-product-id="101">Add to Cart</button>
  </div>
</article>
