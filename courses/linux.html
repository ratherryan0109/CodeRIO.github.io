Linux

Linux is a free, open-source operating system kernel created by Linus Torvalds in 1991. It powers the majority of servers, cloud infrastructure, embedded systems, Android devices, and supercomputers worldwide.

Overview

Linux started as a personal project by Linus Torvalds, a Finnish student who wanted to create a free alternative to MINIX. Today, the Linux kernel is the most widely used open-source software project, with thousands of contributors worldwide. Distributions (distros) like Ubuntu, Debian, Fedora, CentOS, Arch Linux, and Alpine package the kernel with utilities and software for various use cases.

The Linux philosophy emphasizes modularity, simplicity, and the "Unix way" - each tool does one thing well and can be combined with others. The shell (Bash, Zsh) provides powerful scripting capabilities. The filesystem hierarchy standard (FHS) defines the directory structure. Permissions, processes, pipes, and streams form the core of Linux operations.

Linux is the backbone of modern computing. It runs 90%+ of cloud servers, 100% of the top 500 supercomputers, most IoT devices, all Android phones (based on Linux kernel), and countless embedded systems. Mastering Linux is essential for DevOps, cloud engineering, cybersecurity, backend development, and system administration.

Why Learn Linux?

Command Line Power

Automate tasks, process data, manage servers, and control systems efficiently with shell commands and scripting.

Server Dominance

Linux runs the vast majority of web servers, databases, and cloud infrastructure. You can't do server-side work without it.

Security & Stability

Linux is renowned for its security model (user permissions, sudo, AppArmor/SELinux) and can run for years without rebooting.

Essential for DevOps & Cloud

Docker, Kubernetes, Ansible, Terraform, and all major cloud platforms are built on Linux. It's non-negotiable for these roles.

Notes

1. Essential Command Line Operations

The Linux command line is powerful and efficient. Master these fundamental commands for navigating and manipulating the filesystem. Use tab completion, command history (Ctrl+R), and aliases to speed up your workflow.

# Navigation
pwd                      # Print working directory
ls -la                   # List all files with details
cd /var/log              # Change directory
cd ~                     # Go home
cd -                     # Previous directory

# File operations
cp -r /src /dst          # Copy recursively
mv file.txt /tmp/        # Move/rename
rm -rf dir/              # Remove recursively (careful!)
mkdir -p a/b/c           # Create nested directories
touch file.txt           # Create empty file

# Viewing files
cat file.txt             # Display entire file
less file.txt            # Paginated view (q to quit)
head -n 20 file.txt      # First 20 lines
tail -f file.txt         # Follow file (live updates)

2. File Permissions

Every file and directory has three permission sets: owner (u), group (g), and others (o). Permissions are read (r=4), write (w=2), and execute (x=1). Use chmod with octal or symbolic notation.

# View permissions (10 chars)
ls -l script.sh
# -rwxr-xr-- 1 user group 1024 Jan 1 10:00 script.sh
# ^ ^^^ ^^^ ^^^
# | |   |   +--- others: r--
# | |   +------- group: r-x
# | +----------- owner: rwx
# +------------- type: - (file), d (dir), l (link)

# Change with octal
chmod 755 script.sh   # rwxr-xr-x
chmod 644 file.txt    # rw-r--r--
chmod 700 private/    # rwx------

# Change with symbolic
chmod u+x script.sh   # Add execute for owner
chmod go-w file.txt   # Remove write for group/others
chmod a+r file.txt    # Add read for all

# Change owner/group
sudo chown user:group file.txt
sudo chown -R user:group /home/user

3. Pipes, Redirection & Text Processing

Pipes (|) connect commands, sending one command's output as another's input. Redirection (> output, < input, 2> stderr) controls data flow. grep, sed, and awk are the trinity of text processing.

# Pipes - chain commands
ps aux | grep nginx          # Find nginx processes
cat log.txt | grep ERROR | wc -l    # Count errors
dmesg | tail -n 20           # Last kernel messages

# Redirection
echo "hello" > file.txt      # Write (overwrite)
echo "world" >> file.txt     # Append
sort < file.txt              # Input from file
command 2> error.log         # Redirect stderr
command &> all.log           # Redirect both stdout and stderr

# grep: search patterns
grep -i "error" log.txt      # Case-insensitive
grep -r "TODO" src/          # Recursive
grep -v "^#" config.conf     # Exclude comments

# sed: stream editor
sed 's/old/new/g' file.txt   # Replace all
sed -n '10,20p' file.txt     # Print lines 10-20

# awk: text processing
awk '{print $1, $3}' file    # Print columns 1 and 3
awk '/error/ {count++} END {print count}' log.txt

4. Process Management

Every running program is a process with a unique PID. Processes can run in foreground (blocking the terminal) or background. Systemd manages services and system processes. Signals (like SIGTERM, SIGKILL) control processes.

# View processes
ps aux                    # All processes
ps -ef --forest          # Tree view
top                      # Interactive process viewer
htop                     # Enhanced top (install first)
pstree                   # Process tree

# Signals
kill -15 1234            # SIGTERM - graceful shutdown
kill -9 1234             # SIGKILL - force kill
kill -HUP 1234           # SIGHUP - reload config

# Background/Foreground
long-running-command &   # Start in background
Ctrl+Z                   # Suspend (pause) process
bg                       # Resume in background
fg                       # Bring to foreground

# Systemd service management
systemctl status nginx   # Check service status
systemctl start nginx    # Start service
systemctl enable nginx   # Start on boot
journalctl -u nginx -f   # View service logs

5. Shell Scripting

Shell scripts automate repetitive tasks. Bash scripts start with #!/bin/bash shebang. Use variables, conditionals, loops, and functions. Always quote variables to prevent word splitting. Use set -e to exit on errors.

#!/bin/bash
set -euo pipefail

# Variables
NAME="World"
echo "Hello, ${NAME}!"

# Conditionals
if [[ -f "$1" ]]; then
  echo "File exists: $1"
elif [[ -d "$1" ]]; then
  echo "Directory exists: $1"
else
  echo "Not found: $1"
fi

# Loops
for file in *.txt; do
  echo "Processing: $file"
  wc -l "$file"
done

# Functions
backup() {
  local src="$1"
  local dest="/backup/$(basename "$src")-$(date +%Y%m%d)"
  cp -r "$src" "$dest"
  echo "Backup created: $dest"
}

backup "/home/user/documents"

# Exit codes
if ! command -v docker &>/dev/null; then
  echo "Docker not found"
  exit 1
fi

6. Package Management

Package managers install, update, and remove software. Different distributions use different managers but the concepts are the same. Always update the package index before installing. Use packages over manual compilation when possible.

# Debian/Ubuntu (apt)
sudo apt update              # Update package index
sudo apt install nginx       # Install package
sudo apt remove nginx        # Remove package
sudo apt upgrade             # Upgrade all packages
sudo apt autoremove          # Remove unused packages
dpkg -l                      # List installed packages
dpkg -i package.deb          # Install .deb file

# Fedora/RHEL (dnf)
sudo dnf install nginx
sudo dnf remove nginx
sudo dnf groupinstall "Development Tools"
rpm -qa                      # List installed packages

# Arch (pacman)
sudo pacman -S nginx         # Install
sudo pacman -R nginx         # Remove
sudo pacman -Syu             # Full system update

# Universal package formats
snap install nginx           # Snap packages
flatpak install nginx        # Flatpak packages

7. Networking

Linux provides powerful networking tools. Manage network interfaces with ip, troubleshoot with ping/traceroute, test ports with netcat, and download files with curl/wget. Understand the OSI model and TCP/UDP basics.

# Network configuration
ip addr                    # Show IP addresses
ip link                    # Show interfaces
ip route                   # Show routing table
ip link set eth0 up        # Enable interface

# Testing and troubleshooting
ping -c 4 google.com       # Ping with 4 packets
traceroute google.com      # Trace network path
mtr google.com             # Continuous traceroute
ss -tulpn                  # Listening ports
netstat -tulpn             # Alternative to ss

# Downloads
curl -O https://example.com/file.zip
curl -I https://example.com # HTTP headers only
wget -c https://example.com/bigfile.iso  # Resume download

# DNS
dig example.com            # DNS lookup
nslookup example.com       # DNS lookup (legacy)
host example.com           # Simple DNS lookup
/etc/resolv.conf           # DNS configuration
/etc/hosts                 # Local hostname resolution

8. SSH & Remote Access

SSH (Secure Shell) provides encrypted remote access to Linux machines. Use key-based authentication for better security than passwords. SSH can also tunnel traffic, forward ports, and execute commands remotely.

# Generate SSH key pair
ssh-keygen -t ed25519 -C "user@email.com"
# Creates ~/.ssh/id_ed25519 (private) and id_ed25519.pub (public)

# Copy public key to server
ssh-copy-id user@server.com

# Basic SSH
ssh user@server.com                  # Connect
ssh -p 2222 user@server.com          # Custom port
ssh -i ~/.ssh/custom_key user@host   # Specific key

# Execute remote command
ssh user@server.com "df -h; free -m; uptime"

# SCP (secure copy)
scp file.txt user@server.com:/tmp/
scp -r dir/ user@server.com:/tmp/    # Recursive

# RSYNC (efficient sync)
rsync -avz --progress dir/ user@server.com:/backup/

# SSH tunneling (port forwarding)
ssh -L 8080:localhost:80 user@host   # Local port forward
ssh -R 8080:localhost:80 user@host   # Remote port forward

# SSH config (~/.ssh/config)
# Host myserver
#     HostName 192.168.1.100
#     User admin
#     Port 2222
#     IdentityFile ~/.ssh/server_key

Examples

Example 1: System Health Check Script

#!/bin/bash
# system_health.sh - Monitor system resources

set -euo pipefail

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

check_threshold() {
  local value=$1
  local warn=$2
  local crit=$3
  if (( $(echo "$value > $crit" | bc -l) )); then
    echo -e "${RED}CRITICAL${NC}"
  elif (( $(echo "$value > $warn" | bc -l) )); then
    echo -e "${YELLOW}WARNING${NC}"
  else
    echo -e "${GREEN}OK${NC}"
  fi
}

echo "=== System Health Report ==="
echo "Hostname: $(hostname)"
echo "Uptime: $(uptime -p)"
echo

# CPU usage
cpu_idle=$(top -bn1 | grep "Cpu(s)" | awk '{print $8}')
cpu_used=$(echo "100 - $cpu_idle" | bc)
echo -n "CPU Usage: ${cpu_used}% "
check_threshold "$cpu_used" 70 90

# Memory usage
mem_info=$(free | grep Mem)
mem_total=$(echo "$mem_info" | awk '{print $2}')
mem_used=$(echo "$mem_info" | awk '{print $3}')
mem_pct=$(echo "scale=2; $mem_used * 100 / $mem_total" | bc)
echo -n "Memory Usage: ${mem_pct}% "
check_threshold "$mem_pct" 80 95

# Disk usage
df -h | grep '^/dev/' | while read line; do
  disk_pct=$(echo "$line" | awk '{print $5}' | sed 's/%//')
  mount=$(echo "$line" | awk '{print $6}')
  echo -n "Disk ($mount): ${disk_pct}% "
  check_threshold "$disk_pct" 80 95
done

# Top processes by CPU
echo -e "\nTop 5 CPU-consuming processes:"
ps aux --sort=-%cpu | head -6 | tail -5

Example 2: Log Analyzer with awk & grep

#!/bin/bash
# analyze_logs.sh - Quick Nginx/Apache log analyzer

LOG_FILE="${1:-/var/log/nginx/access.log}"

if [[ ! -f "$LOG_FILE" ]]; then
  echo "Log file not found: $LOG_FILE"
  exit 1
fi

echo "=== Log Analysis: $LOG_FILE ==="
echo "Total requests: $(wc -l < "$LOG_FILE")"

echo -e "\nHTTP Status Codes:"
awk '{print $9}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -10

echo -e "\nTop 10 IP addresses:"
awk '{print $1}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -10

echo -e "\nMost requested URLs:"
awk '{print $7}' "$LOG_FILE" | sort | uniq -c | sort -rn | head -10

echo -e "\n404 errors (not found):"
grep ' 404 ' "$LOG_FILE" | awk '{print $7}' | sort | uniq -c | sort -rn | head -10

echo -e "\nRequests by hour:"
awk '{print $4}' "$LOG_FILE" | sed 's/.*://;s/:.*//' | sort | uniq -c | sort -k2

echo -e "\nResponse size summary:"
awk '{sum+=$10; count++} END {
  printf "Total: %.2f MB\n", sum/1024/1024;
  printf "Average: %.2f KB\n", (sum/count)/1024;
}' "$LOG_FILE"

# Real-time monitoring
if [[ "$2" == "--watch" ]]; then
  echo -e "\nWatching for 5xx errors (Ctrl+C to stop)..."
  tail -f "$LOG_FILE" | grep --line-buffered ' 5[0-9][0-9] '
fi
