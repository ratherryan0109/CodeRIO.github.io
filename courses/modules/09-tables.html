HTML Tables

Beginner 50 min Module 9 of 14

Learning Objectives

  • Create tables using <table>, <tr>, <td>, and <th>
  • Use <thead>, <tbody>, and <tfoot> for semantic structure
  • Merge cells with colspan and rowspan
  • Add captions and basic styling to tables

Tables Explained

Think of it like this:

An HTML table is like a spreadsheet in Excel. You have rows (horizontal) and columns (vertical), and each cell contains a piece of data. The <table> is the entire spreadsheet. <tr> (table row) is a horizontal line of cells. <td> (table data) is a single cell. <th> (table header) is like a column header — bold and centered, just like headers in Excel.

Tables are designed for tabular data — information that naturally fits in rows and columns. Think schedules, pricing plans, sports scores, comparison charts, and product specifications. Tables should never be used for page layout (that was common in the 1990s but is now considered bad practice — use CSS Grid or Flexbox instead).

Modern semantic tables use <thead> (header section), <tbody> (main data), and <tfoot> (footer with totals or summaries). These tags do not change the visual appearance, but they help screen readers, search engines, and styling. You can merge cells using colspan (span across columns) and rowspan (span across rows), and add a <caption> to describe the table's purpose.

A table has rows (<tr>) and cells (<td> or <th>).

Beginner Example

<table>
  <tr>
    <th>Product</th>
    <th>Price</th>
    <th>Stock</th>
  </tr>
  <tr>
    <td>Laptop</td>
    <td>$999</td>
    <td>12</td>
  </tr>
  <tr>
    <td>Mouse</td>
    <td>$25</td>
    <td>50</td>
  </tr>
</table>

Line-by-Line Explanation:

  • Lines 2-6: First <tr> contains three <th> elements — column headers. Browsers render them bold and centered.
  • Lines 7-11: Second row of data using <td> (table data) cells. These are regular (not bold) and left-aligned.
  • Lines 12-16: Third row with more data. Each new <tr> adds a horizontal row to the table.

Intermediate Example

<table>
  <caption>Employee Hours Report - Q1 2026</caption>
  <thead>
    <tr>
      <th>Name</th>
      <th>January</th>
      <th>February</th>
      <th>March</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>160</td>
      <td>152</td>
      <td>168</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>144</td>
      <td>160</td>
      <td>155</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th>Total</th>
      <td>304</td>
      <td>312</td>
      <td>323</td>
    </tr>
  </tfoot>
</table>

Line-by-Line Explanation:

  • Line 2: <caption> provides a title for the table. It appears above the table by default and is helpful for screen readers.
  • Lines 4-9: <thead> wraps the header row. This helps browsers and screen readers identify the column headers.
  • Lines 11-23: <tbody> contains the main data rows. You can have multiple <tbody> sections to group data.
  • Lines 24-30: <tfoot> at the bottom with totals. Even if placed before <tbody> in the code, it renders at the bottom.

Advanced Example

<table>
  <caption>Conference Schedule</caption>
  <thead>
    <tr>
      <th rowspan="2">Time</th>
      <th colspan="2">Track 1</th>
      <th colspan="2">Track 2</th>
    </tr>
    <tr>
      <th>Room A</th>
      <th>Room B</th>
      <th>Room C</th>
      <th>Room D</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>9:00 AM</td>
      <td colspan="2">Keynote Address</td>
      <td colspan="2">Workshop Intro</td>
    </tr>
    <tr>
      <td>10:30 AM</td>
      <td>AI Talk</td>
      <td>Web Dev</td>
      <td>Cloud</td>
      <td>Security</td>
    </tr>
  </tbody>
</table>

Line-by-Line Explanation:

  • Line 5: rowspan="2" makes the "Time" cell span two rows vertically — it covers both the header sub-rows.
  • Lines 6, 11: colspan="2" merges two columns. "Track 1" spans Room A and Room B. The heading structure has nested column groups.
  • Line 18: A session that spans both rooms in Track 1 using colspan="2". This represents a joint session in the schedule.

Interview Questions

Q: Why should you not use tables for page layout?

Tables were used for layout in the 1990s, but this is terrible for accessibility (screen readers read tables row by row, mixing up content), responsiveness (tables do not reflow on mobile), and maintainability (nested tables create tangled code). Use CSS Grid or Flexbox for layout and reserve tables for tabular data only.

Q: What is the difference between <thead>, <tbody>, and <tfoot>?

These are semantic elements that group table rows. <thead> contains header rows, <tbody> contains the main data, and <tfoot> contains summary rows (totals, averages). They help with styling (e.g., printing the header/footer on every page), screen reader navigation, and scrolling large tables.

Q: How can you make a table responsive on mobile?

Several approaches: (1) Wrap the table in a <div style="overflow-x: auto"> for horizontal scrolling. (2) Use CSS to restructure cells into a stacked card layout on small screens using display: block and ::before to show labels. (3) Use JavaScript to create an interactive chart from the table data.

Common Mistakes

  • Using tables for layout: Do not put navigation, sidebars, or headers inside tables. Use CSS Grid instead.
  • Mismatched colspan values: If you use colspan="3", the row must have fewer <td> cells to compensate. Mismatched column counts break the table layout.
  • Missing <th> for headers: Header cells should use <th> not <td>. This is important for accessibility so screen readers know which column each cell belongs to.
  • Not using scope on <th>: Add scope="col" or scope="row" to <th> to explicitly associate headers with columns or rows.
