HTML Elements

Beginner 45 min Module 2 of 14

Learning Objectives

  • Distinguish between HTML tags and elements
  • Understand the difference between block-level and inline elements
  • Learn how to properly nest HTML elements
  • Identify void (self-closing) elements and how to use them

Elements Explained

Think of it like this:

An HTML element is like a Russian nesting doll (matryoshka). Each doll can contain smaller dolls inside it. The outermost doll is the <html> element, and it contains the <head> and <body> dolls. Those dolls can contain even more dolls. This concept is called nesting, and it is how HTML builds structure.

An element is the fundamental building block of HTML. Every element consists of an opening tag <tagname>, the content inside, and a closing tag </tagname>. For example, <p>Hello</p> is a paragraph element. The opening tag tells the browser "start a paragraph here," and the closing tag says "stop the paragraph here."

Elements fall into two categories: block-level and inline. Block-level elements (like <div>, <h1>, <p>) start on a new line and take up the full width available. Inline elements (like <span>, <a>, <strong>) sit within the flow of text and only take up as much width as needed. Think of blocks as shipping containers (each on its own truck) and inline elements as items inside a single suitcase.

Beginner Example

<p>This is a paragraph element.</p>
<p>This is another paragraph.</p>
<span>This is inline.</span>
<span>So is this — they sit side by side.</span>

Line-by-Line Explanation:

  • Lines 1-2: Two <p> elements. Each starts on a new line because <p> is a block-level element. You get a visual gap between them automatically.
  • Lines 3-4: Two <span> elements. Both appear on the same line because <span> is inline. It flows with the text like a word in a sentence.

Intermediate Example

<div style="background:lightblue; padding:10px;">
  <h2>My Shopping List</h2>
  <p>Don't forget to buy:</p>
  <ul>
    <li>Milk</li>
    <li>Bread</li>
    <li>Eggs</li>
  </ul>
</div>

Line-by-Line Explanation:

  • Line 1: <div> is a generic block-level container. It groups everything inside it together. The style attribute adds a light blue background and padding for visual grouping.
  • Lines 2-3: A heading and paragraph inside the div. They inherit the div's background and are visually grouped.
  • Lines 4-8: An unordered list <ul> with three list items <li>. This shows nesting — the <ul> is nested inside the <div>, and the <li> elements are nested inside the <ul>.
  • Line 9: The closing </div> closes the container. Everything inside stays grouped together.

Advanced Example

<article>
  <h1>How the Internet Works</h1>
  <p>
    The internet is a <strong>global network</strong> of computers.
    Click <a href="#">here</a> to learn more.
    <br>
    Data travels in <em>packets</em> through routers.
  </p>
  <hr>
  <p>Published on <time datetime="2026-01-15">January 15, 2026</time></p>
</article>

Line-by-Line Explanation:

  • Line 1: <article> — a semantic block element that represents a self-contained piece of content. Think of it like a chapter in a book.
  • Lines 4-5: <strong> and <a> are inline elements nested inside the paragraph. They do not break the text flow — they just wrap specific words.
  • Line 7: <br> is a void element — it inserts a line break and has no closing tag. Void elements are self-contained; they do not wrap any content.
  • Line 8: <em> emphasizes text (usually italic). Also inline.
  • Line 9: <hr> — another void element that draws a horizontal rule (a line) across the page. It acts as a thematic break between sections.
  • Line 10: <time> — a semantic inline element that marks up dates and times in a machine-readable format.

Interview Questions

Q: What is the difference between a block-level element and an inline element?

A block-level element occupies the entire available width and starts on a new line. Examples are <div>, <p>, and <h1>. An inline element only takes up as much width as necessary and flows within the text. Examples are <span>, <a>, and <strong>.

Q: Can you nest any element inside any other element?

No. Some elements have restrictions. For example, a <p> element cannot contain block-level elements like <div> or other <p> tags. Also, void elements like <br> and <hr> cannot contain anything at all since they have no closing tag.

Q: What are void elements? Name three examples.

Void elements (also called self-closing elements) are HTML elements that cannot have any content and do not use a closing tag. They are written as <tagname> or sometimes <tagname />. Examples: <br> (line break), <hr> (horizontal rule), <img> (image), <input> (form input), and <meta> (metadata).

Common Mistakes

  • Crossed nesting: Always close the most recently opened tag first. <div><p>Text</div></p> is wrong; it should be <div><p>Text</p></div>.
  • Block inside inline: You cannot put a <div> inside a <span>. Block elements inside inline elements break the layout because the browser does not know how to handle it.
  • Putting text directly in <ul> or <ol>: List items must be wrapped in <li>. Text directly inside a <ul> without <li> is invalid.
  • Forgetting the closing tag on block elements: Unlike some inline elements, forgetting to close a <div> or <p> can cascade into major layout bugs.
