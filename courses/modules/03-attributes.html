HTML Attributes

Beginner 45 min Module 3 of 14

Learning Objectives

  • Understand what HTML attributes are and how they modify elements
  • Use common attributes like href, src, alt, class, id, and style
  • Learn about boolean attributes and custom data attributes
  • Know how to use global attributes on any element

Attributes Explained

Think of it like this:

If HTML elements are people, then attributes are their accessories and clothes. A person (the element) might wear a name tag (id), a team jersey (class), carry a destination card (href), or have a photo ID (src). The person stays the same, but the accessories tell you more about them and how they behave.

Attributes provide additional information about HTML elements. They always go inside the opening tag and follow the format name="value". Some attributes are required for certain elements to work — for example, an <a> (link) tag needs an href attribute to know where to go, and an <img> tag needs a src attribute to know which image to show.

Attributes can be global (usable on any element, like id, class, and style) or element-specific (only valid for certain elements, like href for <a> or src for <img>). There are also boolean attributes like disabled or required that do not need a value — just their presence means "true."

Beginner Example

<a href="https://google.com">Go to Google</a>
<img src="photo.jpg" alt="A sunny beach">
<p class="highlight">This paragraph has a class.</p>
<div id="header">This div has a unique ID.</div>

Line-by-Line Explanation:

  • Line 1: href="https://google.com" is an attribute on the <a> tag. It tells the browser where the link should take you when clicked.
  • Line 2: src="photo.jpg" tells the browser which image file to load. alt="A sunny beach" provides a text description — it shows if the image fails to load and helps screen readers.
  • Line 3: class="highlight" assigns a class name that CSS can target. Multiple elements can share the same class.
  • Line 4: id="header" gives this div a unique identifier. No two elements on a page should have the same id.

Intermediate Example

<button disabled>Click Me (Disabled)</button>
<input type="text" placeholder="Enter your name" required>
<a href="#section2">Jump to Section 2</a>
<img src="logo.svg" alt="Company logo" width="200" height="100">

Line-by-Line Explanation:

  • Line 1: disabled is a boolean attribute. Just writing disabled (without a value) disables the button. You could also write disabled="disabled".
  • Line 2: type="text" specifies the input type. placeholder shows hint text inside the field. required is another boolean attribute — the form will not submit unless this field is filled in.
  • Line 3: href="#section2" creates a link to an element with id="section2" on the same page. The # symbol means "within this page."
  • Line 4: width="200" height="100" sets the image dimensions. Always include these so the browser reserves space before the image loads, preventing layout shifts.

Advanced Example

<article class="post featured" id="post-101" data-author="alex" data-published="2026-01-15">
  <h2 title="Click to read more">10 HTML Tips</h2>
  <p lang="fr">Ceci est un paragraphe en français.</p>
  <button accesskey="s" tabindex="1" aria-label="Save changes">Save</button>
  <div contenteditable="true" spellcheck="true">
    You can edit this text directly.
  </div>
</article>

Line-by-Line Explanation:

  • Line 1: The article has two classes (post featured) separated by a space. data-author and data-published are custom data attributes — you can invent any data-* attribute to store extra information for JavaScript.
  • Line 2: title="Click to read more" creates a tooltip that appears when you hover over the heading. It is a global attribute available on any element.
  • Line 3: lang="fr" tells the browser this paragraph is in French. Screen readers use this to switch pronunciation.
  • Line 4: accesskey="s" defines a keyboard shortcut (Alt+S). tabindex="1" controls the order when pressing Tab. aria-label provides an accessible name for screen readers.
  • Line 5: contenteditable="true" makes the div editable like a text field. spellcheck="true" enables spell-checking inside it.

Build a "Product Card" using HTML attributes effectively. Create a <div> with class="product-card" and data-product-id="001". Inside, add an image with src, alt, width, and height. Hints:

  • Use target="_blank" if you add any external links so they open in a new tab.
  • Boolean attributes like disabled do not need an equals sign or value — just write the attribute name.
  • Custom data attributes must always start with data- followed by a lowercase name.

Interview Questions

Q: What is the difference between id and class attributes?

An id must be unique on the page — you cannot have two elements with the same id. It is used for targeting a single element with CSS, JavaScript, or anchor links. A class can be reused on multiple elements to apply the same styling or behavior to a group.

Q: What are data attributes and how are they used?

Data attributes (data-*) allow you to store custom information on any HTML element. They are accessed in JavaScript via the dataset property. For example, element.dataset.userId reads the data-user-id attribute. They are perfect for storing state or identifiers without using hidden inputs or classes.

Q: What does the alt attribute on an image do and why is it important?

The alt attribute provides alternative text for an image. It is displayed if the image fails to load, read aloud by screen readers for visually impaired users, and used by search engines to understand the image content. It is required for accessibility (WCAG) and improves SEO.

Common Mistakes

  • Duplicate IDs: Using the same id value on multiple elements. IDs must be unique per page. Use class instead for repeated values.
  • Missing quotes around attribute values: href=https://example.com works in some browsers but fails if the URL contains special characters. Always use quotes.
  • Forgetting the alt attribute on images: Every <img> tag must have an alt attribute for accessibility. If the image is purely decorative, use alt="" (empty).
  • Spaces in attribute values without quotes: class=highlight text is parsed as two attributes. Use class="highlight text" to make it a single value.
