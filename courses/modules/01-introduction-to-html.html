Introduction to HTML

Beginner 45 min Module 1 of 14

Learning Objectives

  • Understand what HTML is and why it is essential for the web
  • Recognize the basic structure of an HTML document
  • Write your first HTML page with headings and paragraphs
  • Learn how tags, elements, and attributes work together

HTML Explained

Think of it like this:

Imagine you are building a house. The HTML is the skeleton — the wooden frame that defines where the walls, windows, and doors go. CSS is the paint and wallpaper, and JavaScript is the electricity and plumbing that makes things happen. Without the skeleton, you have nothing to attach the other layers to.

HTML stands for HyperText Markup Language. It is the standard language used to create web pages. Every single website you visit — from Google to YouTube to this very page — is built with HTML at its core. When you "view page source" in your browser, you are looking at the raw HTML that structures the page.

HTML is not a programming language — it is a markup language. That means you use it to describe and label content, not to write logic or calculations. You tell the browser "this is a heading," "this is a paragraph," "this is a link," and the browser knows how to display it. Think of it like giving stage directions to an actor rather than writing the script of a play.

Visual Diagram

<html> — The entire webpage
<head>
Metadata, title, links
<body>
Visible content: headings, paragraphs, images, links

Every HTML page has two main parts: the head (invisible setup) and the body (what you see).

Beginner Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My First Page</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>This is my first HTML page.</p>
</body>
</html>

Line-by-Line Explanation:

  • Line 1: <!DOCTYPE html> — tells the browser this is an HTML5 document. It is like saying "this is a recipe book" before you open it.
  • Line 2: <html lang="en"> — the root element that wraps everything. lang="en" tells the browser the page is in English.
  • Lines 3-5: The <head> — contains invisible setup: character encoding and the page title shown on the browser tab.
  • Lines 6-9: The <body> — everything here appears on the screen. <h1> is a main heading, and <p> is a paragraph.
  • Line 10: </html> — closes the root element. Always close what you open.

Intermediate Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>About Me</title>
</head>
<body>
  <h1>Hi, I'm Alex</h1>
  <h2>A Web Developer in Training</h2>
  <p>I love building things for the web. HTML is my first step.</p>
  <p>When I am not coding, I enjoy hiking and reading.</p>
</body>
</html>

Line-by-Line Explanation:

  • Lines 4-5: Two meta tags: charset ensures special characters (like é or ü) display correctly; viewport makes the page look good on phones.
  • Line 10: <h2> is a sub-heading. You can have multiple heading levels (h1 through h6).
  • Lines 11-12: Two separate <p> tags create two paragraphs. The browser adds space between them automatically.

Advanced Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="description" content="A personal portfolio showcasing web projects">
  <title>Alex Rivera | Portfolio</title>
</head>
<body>
  <header>
    <h1>Alex Rivera</h1>
    <nav>
      <a href="#about">About</a>
      <a href="#projects">Projects</a>
      <a href="#contact">Contact</a>
    </nav>
  </header>
  <main>
    <section id="about">
      <h2>About Me</h2>
      <p>I build accessible, performant websites.</p>
    </section>
    <section id="projects">
      <h2>Projects</h2>
      <p>Coming soon!</p>
    </section>
  </main>
  <footer>
    <p>&copy; 2026 Alex Rivera</p>
  </footer>
</body>
</html>

Line-by-Line Explanation:

  • Line 5: A meta description tag — this is the snippet search engines show under your page title. Great for SEO.
  • Lines 10-16: The <header> element contains introductory content and a <nav> with navigation links. Semantic HTML helps search engines understand your page.
  • Lines 17-24: The <main> element wraps the primary content. Inside it, <section> elements group related content by topic.
  • Line 25: <footer> holds copyright info. The &copy; is an HTML entity that renders the © symbol.

Your page must include: a DOCTYPE declaration, an html tag with lang attribute, a head with charset and title meta tags, a body with at least one h1 heading, two h2 sub-headings, three paragraphs, and a footer.   • Use Ctrl+S (or Cmd+S on Mac) to save your file, then double-click it to open in your browser.
  • If your changes are not showing up, try a hard refresh with Ctrl+Shift+R.

Interview Questions

Q: What is the purpose of the <!DOCTYPE html> declaration?

It tells the browser which version of HTML the page is written in. For HTML5, it is simply <!DOCTYPE html>. Without it, some browsers may render the page in "quirks mode," which can cause layout issues.

Q: What is the difference between a tag and an element?

A tag is the markup syntax like <p> or </p>. An element includes the opening tag, the content, and the closing tag together. So <p>Hello</p> is an element made of two tags and the content "Hello".

Q: Why is the <head> section important even though nothing inside it shows on the page?

The head contains critical metadata: the page title (shown on the browser tab), character encoding, viewport settings for mobile devices, links to CSS stylesheets, and meta tags for SEO. Without these, your page may not display correctly or rank well in search engines.

Common Mistakes

  • Forgetting to close tags: Every opening tag (except void elements) needs a closing tag. <p>Text without </p> can break your layout.
  • Using uppercase tags: While <HTML> technically works, the convention is lowercase. Stick with <html> for consistency and readability.
  • Putting content inside the head: Anything you want users to see must go inside <body>. The head is only for setup.
  • Nesting incorrectly: Tags must close in reverse order of opening. <p><strong>Text</p></strong> is wrong — it should be <p><strong>Text</strong></p>.
