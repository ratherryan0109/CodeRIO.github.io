Semantic HTML

Beginner 50 min Module 11 of 14

Learning Objectives

  • Understand what semantic HTML means and why it matters
  • Use semantic elements like <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
  • Replace generic <div> elements with meaningful semantic tags
  • Learn how semantic HTML improves accessibility, SEO, and code readability

Semantic HTML Explained

Think of it like this:

Imagine receiving a package with no labels — just a plain brown box. You have no idea what is inside or where it came from. That is what non-semantic HTML (<div> and <span> everywhere) is like. Now imagine a box clearly labeled "FRAGILE — ELECTRONICS" with a return address and a contents list. That is semantic HTML — every element tells you something about its content. The browser, search engine, and screen reader all understand what each part of your page means.

Semantic HTML means using HTML elements that describe their meaning rather than just their appearance. For example, <nav> means "this is navigation," <article> means "this is a self-contained piece of content," and <footer> means "this is the footer of the page or section." Compare this to the generic <div> which means nothing — it is just a container. Semantic elements tell a story about your page structure.

Why does this matter? Accessibility: Screen readers use semantic elements to help users navigate. A visually impaired user can jump from <nav> to <main> to <footer> without hearing every word. SEO: Search engines give more weight to content inside <article> and <main> and use <nav> to understand your site structure. Readability: Other developers can look at your HTML and immediately understand the page layout without reading every line.

Beginner Example

<header>
  <h1>My Blog</h1>
  <p>Thoughts on web development</p>
</header>

<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>

<main>
  <article>
    <h2>Why I Love HTML</h2>
    <p>HTML is the foundation of the web...</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 My Blog</p>
</footer>

Line-by-Line Explanation:

  • Lines 1-4: <header> — introductory content for the page. Usually contains the site title, logo, and tagline.
  • Lines 6-10: <nav> — a section with navigation links. Screen readers immediately announce "navigation" when reaching this element.
  • Lines 12-16: <main> — the primary content of the page. There should be only one <main> per page. Inside it, <article> represents a self-contained piece of content.
  • Lines 18-20: <footer> — contains copyright, contact info, or related links. Often repeated across pages.

Intermediate Example

<body>
  <header>
    <h1>TechReview</h1>
    <nav>
      <a href="/">Home</a>
      <a href="/reviews">Reviews</a>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>iPhone 16 Review</h2>
        <time datetime="2026-03-15">March 15, 2026</time>
      </header>
      <section>
        <h3>Design</h3>
        <p>The phone features a titanium frame...</p>
      </section>
      <section>
        <h3>Camera</h3>
        <p>The new 48MP sensor captures stunning photos...</p>
      </section>
    </article>

    <aside>
      <h3>Related Articles</h3>
      <a href="#">Best Phones of 2026</a>
      <a href="#">iOS 20 Features</a>
    </aside>
  </main>

  <footer>
    <p>&copy; 2026 TechReview</p>
  </footer>
</body>

Line-by-Line Explanation:

  • Lines 12-15: An <article> with its own <header> containing the article title and publication date using <time>.
  • Lines 16-23: <section> elements divide the article into logical parts (Design, Camera). Each section has its own <h3> heading.
  • Lines 25-29: <aside> contains content indirectly related to the main content — sidebars, related links, advertisements.

Advanced Example

<body>
  <header>
    <hgroup>
      <h1>The Dev Daily</h1>
      <p>A developer's guide to modern web</p>
    </hgroup>
    <nav aria-label="Main navigation">
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/tutorials">Tutorials</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article aria-labelledby="article-title">
      <h2 id="article-title">Understanding Flexbox</h2>
      <p>Flexbox is a CSS layout module...</p>
      <section aria-label="Code Example">
        <h3>Basic Flexbox Setup</h3>
        <pre><code>.container { display: flex; }</code></pre>
      </section>
    </article>

    <aside>
      <nav aria-label="Table of contents">
        <h3>In This Article</h3>
        <ol>
          <li><a href="#intro">Introduction</a></li>
          <li><a href="#setup">Setup</a></li>
        </ol>
      </nav>
    </aside>
  </main>

  <footer>
    <address>
      Contact: <a href="mailto:hello@devdaily.com">hello@devdaily.com</a>
    </address>
    <small>&copy; 2026 The Dev Daily</small>
  </footer>
</body>

Line-by-Line Explanation:

  • Lines 3-6: <hgroup> groups multiple heading levels (h1 and p) as a single heading for the page.
  • Lines 8, 29, 32: aria-label attributes provide accessible names for <nav> elements when there are multiple. This helps screen readers distinguish between "Main navigation" and "Table of contents."
  • Line 30: <address> — a semantic element for contact information. It typically renders in italic.
  • Line 31: <small> — represents side-comments like fine print or copyright.

Assignment

Build a full "News Article" page using only semantic HTML elements (no divs or spans unless absolutely necessary). Your page must include: a header with site name and nav, a main area containing an article with a header (title + date), at least 3 sections with h3 headings, an aside with related links, and a footer with copyright and address. Add ARIA labels where you have multiple navigations. Add a figure with figcaption inside one of the sections. Validate your page with the W3C Nu HTML Checker. Save as news-article.html.

Hints:

  • Use the W3C Nu Checker (validator.w3.org/nu/) to validate your HTML and check for semantic correctness.
  • You can nest <header> inside <article> — each sectioning element can have its own header.
  • Use <figure> and <figcaption> for images with captions inside sections.

Interview Questions

Q: What is the difference between <div> and <section>?

A <div> has no semantic meaning — it is a generic container for styling or scripting. A <section> represents a thematic grouping of content, typically with a heading. Use <section> when the content forms a distinct part of the document outline. Use <div> only when no other semantic element fits.

Q: Can you use multiple <header> or <footer> elements on one page?

Yes. Each sectioning element (<article>, <section>, <aside>, <nav>) can have its own <header> and <footer>. For example, an <article> can have a header with its title and a footer with its publication date and author.

Q: What is the difference between <article> and <section>?

An <article> is a self-contained composition that could be distributed independently (a blog post, news story, forum post). A <section> is a thematic grouping within a document. An <article> can contain multiple <section> elements, and a <section> can contain multiple <article> elements. Choose based on whether the content makes sense on its own.

Common Mistakes

  • Using <div> for everything: Non-semantic "div soup" makes your code unreadable and inaccessible. Always ask "is there a better element for this?"
  • Putting <nav> inside <header> unnecessarily: While common, a <nav> can exist outside the <header> — for example, a breadcrumb nav in the <main> area.
  • Using <main> multiple times: There should be only one <main> element per page. It should not be inside other sectioning elements.
  • Missing heading hierarchy in sections: Each <section> should ideally have a heading (<h1> to <h6>) to describe its content.
