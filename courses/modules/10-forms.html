HTML Forms

Beginner 55 min Module 10 of 14

Learning Objectives

  • Create forms using the <form> element with action and method
  • Use various input types: text, email, password, number, checkbox, radio
  • Add labels, placeholders, and validation attributes
  • Use textarea, select/dropdown, and button elements

Forms Explained

Think of it like this:

An HTML form is like a paper application form at a doctor's office. There are different fields for different types of information: text fields for your name, checkboxes for symptoms, radio buttons for gender, dropdowns for your state, and a big text area for additional notes. When you fill it out and click "Submit," the form sends all that data to the office's computer system (the server).

Forms are how users interact with websites. Logging in, signing up, searching, commenting, purchasing, uploading — almost every interactive feature involves a form. The <form> element wraps all the inputs. The action attribute specifies where to send the data (a URL), and the method attribute specifies how to send it (GET for visible data in the URL, POST for hidden data in the request body).

Every input should have a matching <label> — this is critical for accessibility (screen readers announce the label when the input is focused) and usability (clicking the label focuses the input). HTML5 introduced many input type values that enable browser-level validation: email, url, number, tel, date, color, range, and more. The browser automatically checks the format and shows error messages for invalid data.

Beginner Example

<form action="/submit" method="POST">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" placeholder="Your full name">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" placeholder="you@example.com">

  <button type="submit">Subscribe</button>
</form>

Line-by-Line Explanation:

  • Line 1: <form> element: action="/submit" = send data to this URL; method="POST" = send data hidden in the request body (not in the URL).
  • Lines 2, 5: <label for="..."> — the for attribute must match the input's id. Clicking the label focuses the input field.
  • Lines 3, 6: type="text" is a simple text field. type="email" tells the browser to validate that the input is a valid email format.
  • Line 8: <button type="submit"> submits the form. Without type="submit", a button inside a form submits by default anyway.

Intermediate Example

<form action="/signup" method="POST">
  <label for="username">Username:</label>
  <input type="text" id="username" name="username" required minlength="3">

  <label for="password">Password:</label>
  <input type="password" id="password" name="password" required minlength="8">

  <fieldset>
    <legend>Gender</legend>
    <input type="radio" id="male" name="gender" value="male">
    <label for="male">Male</label>
    <input type="radio" id="female" name="gender" value="female">
    <label for="female">Female</label>
  </fieldset>

  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="">-- Select --</option>
    <option value="us">United States</option>
    <option value="uk">United Kingdom</option>
    <option value="ca">Canada</option>
  </select>

  <label>
    <input type="checkbox" name="terms" required>
    I agree to the terms and conditions
  </label>

  <button type="submit">Create Account</button>
</form>

Line-by-Line Explanation:

  • Line 3: required — the form will not submit if this field is empty. minlength="3" — requires at least 3 characters.
  • Line 6: type="password" — masks the input with dots for privacy. The data is still sent to the server.
  • Line 8: <fieldset> groups related inputs. <legend> provides a label for the group. This is great for radio button groups.
  • Lines 9-12: Radio buttons share the same name attribute, so only one can be selected at a time.
  • Lines 16-21: <select> creates a dropdown menu. Each <option> is an item. The first option is a placeholder.
  • Lines 23-25: A checkbox wrapped inside a <label> — this way clicking the text toggles the checkbox.

Advanced Example

<form action="/feedback" method="POST">
  <label for="rating">Rating: <span id="ratingValue">5</span></label>
  <input type="range" id="rating" name="rating" min="1" max="10" value="5"
         oninput="document.getElementById('ratingValue').textContent = this.value">

  <label for="message">Your Feedback:</label>
  <textarea id="message" name="message" rows="5" cols="40"
            placeholder="Share your thoughts..." maxlength="500"></textarea>

  <input type="hidden" name="userId" value="12345">

  <datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Edge">
    <option value="Safari">
  </datalist>
  <label for="browser">Browser:</label>
  <input type="text" id="browser" name="browser" list="browsers">

  <button type="submit">Send Feedback</button>
</form>

Line-by-Line Explanation:

  • Lines 3-4: type="range" creates a slider. The oninput JavaScript updates the displayed value in real-time as the user slides.
  • Lines 7-8: <textarea> — a multi-line text input. rows and cols set the visible size. maxlength="500" limits character count.
  • Line 10: type="hidden" — this field is invisible to the user but sends data with the form. Useful for tracking IDs or tokens.
  • Lines 12-18: <datalist> provides autocomplete suggestions. The input with list="browsers" shows suggestions as the user types, but they can also type a custom value.

Interview Questions

Q: What is the difference between GET and POST methods in forms?

GET appends form data to the URL as query parameters (visible in the address bar). It is suitable for search forms and idempotent actions. POST sends data in the request body (not visible in the URL). It is used for actions that change data (signups, logins, purchases) and can handle larger amounts of data.

Q: Why is it important to associate labels with inputs using the "for" attribute?

It improves accessibility — screen readers announce the label when the input is focused. It also improves usability — clicking the label text focuses/activates the input, which is especially helpful for small checkboxes and radio buttons. It also increases the clickable area on touch devices.

Q: What are the different types of input buttons?

There are three: <input type="submit"> submits the form, <input type="reset"> resets all fields to their default values, and <input type="button"> does nothing by default (used with JavaScript). The <button> element is more flexible as it can contain HTML content (icons, images).

Common Mistakes

  • Forgetting the name attribute: The name attribute is what the server uses to identify each field. Without it, the data is not sent. The id is only for labels and CSS/JS.
  • Not using labels: Every input needs a label. Never use placeholder as a substitute for a label — placeholders disappear when the user starts typing and fail accessibility checks.
  • Nested forms: You cannot put a <form> inside another <form>. The browser ignores the inner form.
  • Forgetting to set input types: Using type="text" for everything misses out on browser features like email validation, number spinners, and mobile-friendly keyboards.
