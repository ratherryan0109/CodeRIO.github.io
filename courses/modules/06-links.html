HTML Links

Beginner 45 min Module 6 of 14

Learning Objectives

  • Create hyperlinks using the <a> tag with the href attribute
  • Open links in new tabs with target="_blank"
  • Create email links with mailto: and phone links with tel:
  • Use bookmark links to navigate within the same page

Links Explained

Think of it like this:

A link is like a wormhole in space. You start on one page (planet), click a link, and instantly travel to another page (distant galaxy). The <a> tag creates the wormhole, and the href attribute is the coordinates it connects to. Without href, the link is like a broken wormhole — it looks like it should work, but clicking it does nothing.

The <a> (anchor) tag is the foundation of the web. Without links, the web would be a collection of isolated pages — there would be no "surfing," no "browsing," no interconnected information. The href (hypertext reference) attribute tells the browser where the link goes. It can point to another webpage, a file, an email address, a phone number, or even a specific section on the same page.

Links have different states that users can recognize: unvisited (default blue, underlined), visited (purple), hover (when the mouse is over it), and active (while being clicked). You can style these with CSS. Links are inline elements, but they can contain block-level children (like an entire <div>) in HTML5 — making clickable cards and panels easy to build.

Beginner Example

<a href="https://google.com">Visit Google</a>
<a href="about.html">About Us (same site)</a>
<a href="#contact">Jump to Contact Section</a>
<a href="mailto:hello@example.com">Email Us</a>

Line-by-Line Explanation:

  • Line 1: Full URL (absolute link) — takes you to Google's homepage. The https:// part is required for external URLs.
  • Line 2: Relative link — points to about.html in the same folder. No domain needed because it is on the same site.
  • Line 3: Bookmark link — the # means "look for an element with id="contact" on this page."
  • Line 4: mailto: link — opens the user's default email client with hello@example.com in the To field.

Intermediate Example

<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  Open in New Tab <i class="fas fa-external-link-alt"></i>
</a>
<a href="tel:+1234567890">Call Us: (123) 456-7890</a>
<a href="document.pdf" download>Download PDF</a>
<a href="#top">Back to Top</a>

Line-by-Line Explanation:

  • Line 1: target="_blank" opens the link in a new tab. rel="noopener noreferrer" is a security measure — it prevents the new page from accessing your page via JavaScript.
  • Line 4: tel:+1234567890 — on mobile devices, this prompts the user to call the number. On desktop, it may open a calling app.
  • Line 5: The download attribute tells the browser to download the linked file instead of navigating to it. No value needed — it is a boolean attribute.
  • Line 6: #top is a special bookmark that scrolls to the very top of the page. Alternatively, you could use # for the same effect.

Advanced Example

<a href="https://maps.google.com?q=Times+Square" title="Open in Google Maps">
  <div class="map-card">
    <img src="map-thumb.jpg" alt="Map thumbnail">
    <h3>Times Square</h3>
    <p>New York, NY 10036</p>
  </div>
</a>

<a href="mailto:support@company.com?subject=Help%20Request&body=Describe%20your%20issue">
  Send Support Email
</a>

Line-by-Line Explanation:

  • Lines 1-7: An anchor wrapping an entire card (div, image, heading, paragraph). In HTML5, this is valid — clicking anywhere on the card follows the link. The title attribute adds a tooltip on hover.
  • Lines 9-10: A mailto: link with pre-filled subject and body. The ? separates the email address from the parameters. & separates multiple parameters. Spaces are encoded as %20.

Interview Questions

Q: What is the difference between absolute and relative URLs?

An absolute URL includes the full path with protocol and domain, e.g., https://example.com/page.html. It works from anywhere. A relative URL is a path relative to the current page, e.g., page.html or ../folder/page.html. Relative URLs are shorter and make your site portable between domains.

Q: What is the security issue with target="_blank" without rel="noopener"?

Without rel="noopener", the newly opened page can access your page via the window.opener object. This is a tabnabbing vulnerability — the malicious page can redirect your page to a phishing site. rel="noopener" prevents this by setting window.opener to null.

Q: Can an <a> tag contain block-level elements?

In HTML5, yes. The <a> element can wrap block-level elements like <div>, <h1>, or <p>. This allows creating entire clickable cards. However, you cannot nest interactive elements (like <button> or other <a> tags) inside an anchor.

Common Mistakes

  • Forgetting https:// in external links: href="google.com" tries to open yoursite.com/google.com (relative). Always include the protocol: href="https://google.com".
  • Using target="_blank" without rel="noopener": This creates a security vulnerability. Always pair them: target="_blank" rel="noopener noreferrer".
  • Broken bookmark anchors: A link to #contact will not work unless there is an element with id="contact" on the page.
  • Vague link text: "Click here" is bad for SEO and accessibility. Screen reader users often navigate by listing links — "Click here" tells them nothing. Use descriptive text like "View our pricing page."
