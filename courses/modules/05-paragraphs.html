HTML Paragraphs

Beginner 40 min Module 5 of 14

Learning Objectives

  • Use the <p> tag to create paragraphs
  • Add line breaks with <br> and horizontal rules with <hr>
  • Preserve whitespace and formatting with <pre>
  • Understand how the browser collapses whitespace in paragraphs

Paragraphs Explained

Think of it like this:

A paragraph in HTML is like a paragraph in a book. It is a block of text that forms a complete thought. The <p> tag wraps that block. Just as a book has paragraphs separated by blank lines, your HTML paragraphs automatically get spacing between them. If you want a new line within the same paragraph (like a line in a poem), you use <br> — imagine that as pressing Shift+Enter in a word processor instead of Enter.

The <p> tag is one of the most commonly used HTML elements. It is a block-level element, meaning each paragraph starts on a new line and takes the full width available. Browsers automatically add margin above and below paragraphs to create visual separation. One common gotcha: HTML collapses whitespace — if you write ten spaces between words or add multiple blank lines in your code, the browser will only show one space. This is called whitespace collapsing.

To control text layout more precisely, HTML gives you two helper elements: <br> (line break) and <hr> (horizontal rule). The <br> tag forces a line break without starting a new paragraph — useful for addresses, poems, or song lyrics. The <hr> tag draws a thematic line across the page, signaling a shift in topic. Both are void elements (no closing tag). For text where spacing matters exactly as written, use <pre> (preformatted text) — it preserves every space, tab, and line break.

Beginner Example

<p>This is a paragraph. It can contain multiple sentences. The browser will wrap the text automatically when it reaches the edge of the screen.</p>
<p>This is another paragraph. Notice the space between them.</p>
<p>This line has a <br> line break in the middle.</p>

Line-by-Line Explanation:

  • Lines 1-2: Two separate <p> elements. The browser adds vertical space between them automatically — no need for extra <br> tags.
  • Line 3: A <br> inside a paragraph forces the text after it to the next line, but it stays within the same paragraph (no extra paragraph spacing).

Intermediate Example

<p>Here is an address:</p>
<p>
  John Doe<br>
  123 Main Street<br>
  New York, NY 10001<br>
  USA
</p>
<hr>
<p>This section discusses our services.</p>

Line-by-Line Explanation:

  • Lines 4-7: An address formatted with <br> after each line. Each line is part of the same paragraph but appears on its own line.
  • Line 8: <hr> draws a horizontal line across the page and creates a thematic break between the address and the next section.
  • Line 9: A new paragraph after the hr. The hr provides visual separation without needing to start a new section.

Advanced Example

<p>In JavaScript, you can define a function like this:</p>
<pre><code>function greet(name) {
    console.log("Hello, " + name + "!");
    return true;
}
</code></pre>
<hr>
<p>Poem:</p>
<p>
  The woods are lovely, dark and deep,<br>
  But I have promises to keep,<br>
  And miles to go before I sleep,<br>
  And miles to go before I sleep.
</p>

Line-by-Line Explanation:

  • Lines 3-7: <pre> preserves the indentation, spaces, and line breaks exactly as written. The <code> inside marks it as computer code. Without <pre>, the indentation would be collapsed.
  • Line 9: <hr> separates the code example from the poem — two completely different types of content.
  • Lines 11-14: A poem using <br> for line breaks inside a single paragraph. Each line break is intentional and keeps the poem as one semantic block of text.

Interview Questions

Q: What is the difference between <br> and <p> for creating space between lines?

<br> creates a line break within the same paragraph — it adds a new line but no extra spacing. <p> creates a new paragraph with top and bottom margins (spacing) applied by the browser. Use <p> for separate blocks of text and <br> for single-line breaks inside a block.

Q: Why does my text not show multiple spaces even though I typed them in HTML?

HTML collapses whitespace — any sequence of spaces, tabs, or newlines in the source code is rendered as a single space in the browser. This is by design to let you format your code without affecting the visual output. Use &nbsp; (non-breaking space) or <pre> if you need to preserve multiple spaces.

Q: When should I use <hr>?

Use <hr> to indicate a thematic shift between sections of content. For example, between chapters in a story, between a blog post and its comments, or between different topics on a page. It is semantic, not just decorative. Do not use it purely for styling — that is what CSS borders are for.

Common Mistakes

  • Using <br> to create space between paragraphs: Instead of <p>A</p><br><br><p>B</p>, just use two separate <p> tags. The browser adds spacing automatically.
  • Forgetting that <br> is a void element: Writing <br></br> is wrong. Just use <br>.
  • Relying on <hr> for decoration: The <hr> tag is semantic — it represents a thematic break. For decorative lines, use CSS borders instead.
  • Assuming <pre> is only for code: While <pre> is great for code, it can also display ASCII art, formatted letters, or any text where whitespace matters.
