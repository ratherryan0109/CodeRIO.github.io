HTML Lists

Beginner 40 min Module 8 of 14

Learning Objectives

  • Create unordered lists with <ul> and <li>
  • Create ordered lists with <ol> and <li>
  • Use description lists with <dl>, <dt>, and <dd>
  • Nest lists inside other lists for complex structures

Lists Explained

Think of it like this:

HTML lists are like different types of shopping lists. An unordered list (<ul>) is a grocery list — the order of items does not matter, so each gets a bullet point. An ordered list (<ol>) is a recipe step list — order is critical, so each step gets a number. A description list (<dl>) is like a dictionary or FAQ — each term has a definition below it.

Lists are everywhere on the web: navigation menus, ingredient lists, top 10 rankings, FAQs, comment threads, and product features. The three list types cover almost every scenario. All lists are block-level and consist of a container element and child items. The <li> (list item) element is used for both <ul> and <ol>.

Lists can be nested — placing one list inside an <li> of another list creates sub-lists. This is useful for outlines, multi-level navigation, or categorized items. You can also customize the bullet style for <ul> using the list-style-type CSS property (disc, circle, square, none) and the numbering style for <ol> (decimal, roman, alpha).

Beginner Example

<h2>Grocery List</h2>
<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Eggs</li>
  <li>Apples</li>
</ul>

<h2>Top 3 Movies</h2>
<ol>
  <li>The Matrix</li>
  <li>Inception</li>
  <li>Interstellar</li>
</ol>

Line-by-Line Explanation:

  • Lines 2-7: <ul> creates a bulleted list. Each <li> becomes a bullet point. The order does not matter visually.
  • Lines 10-14: <ol> creates a numbered list. The browser automatically numbers each <li> starting from 1.

Intermediate Example

<h2>Course Syllabus</h2>
<ol type="I">
  <li>Introduction
    <ul>
      <li>Welcome & setup</li>
      <li>First HTML page</li>
    </ul>
  </li>
  <li>HTML Basics
    <ul>
      <li>Tags & elements</li>
      <li>Attributes</li>
    </ul>
  </li>
</ol>

Line-by-Line Explanation:

  • Line 2: type="I" changes the numbering to uppercase Roman numerals (I, II, III). You can also use a, A, i, or 1.
  • Lines 4-7: A <ul> nested inside an <li>. This creates a sub-list under "Introduction." The nested list is indented and uses a different bullet style.
  • Lines 10-13: The same nesting pattern repeats. Each main topic has sub-topics in a bullet list.

Advanced Example

<h2>Web Development FAQ</h2>
<dl>
  <dt>What is HTML?</dt>
  <dd>HyperText Markup Language — the structure of web pages.</dd>

  <dt>What is CSS?</dt>
  <dd>Cascading Style Sheets — controls the visual presentation.</dd>

  <dt>What is JavaScript?</dt>
  <dd>A programming language that adds interactivity to websites.</dd>
</dl>

<h2>Multi-level Navigation</h2>
<ul class="nav-menu">
  <li>Home
    <ul>
      <li>Dashboard</li>
      <li>Activity</li>
    </ul>
  </li>
  <li>Settings
    <ul>
      <li>Profile
        <ul>
          <li>Edit Avatar</li>
          <li>Change Password</li>
        </ul>
      </li>
      <li>Privacy</li>
    </ul>
  </li>
</ul>

Line-by-Line Explanation:

  • Lines 3-10: <dl> (description list) pairs terms (<dt>) with definitions (<dd>). Browsers typically indent the <dd> text. This is perfect for glossaries, FAQs, or metadata.
  • Lines 14-31: Three levels of nesting: main menu → sub-menu → sub-sub-menu. Each level is indented further and uses a different bullet style automatically.
  • Line 16: <li>Home — the text "Home" is before the nested <ul>. This creates a parent item with expandable children, the foundation of dropdown menus.

  • Use <ol start="10"> if you want numbering to start from a specific number.

Interview Questions

Q: When should you use <ul> vs <ol>?

Use <ul> (unordered list) when the order of items does not matter — feature lists, navigation menus, ingredients. Use <ol> (ordered list) when sequence is important — step-by-step instructions, rankings, timelines. Semantically choosing the right list type helps screen readers convey the correct meaning.

Q: What is the purpose of the <dl> element?

The <dl> (description list) element is used to group terms and their descriptions. It is ideal for glossaries, FAQs, dictionaries, or any key-value pair data. Each term uses <dt> and each definition uses <dd>. One <dt> can have multiple <dd>s and vice versa.

Q: How can you change the bullet style of an unordered list?

Use the CSS list-style-type property. Common values: disc (default filled circle), circle (hollow circle), square, and none. You can also use custom images with list-style-image: url('bullet.png'). For ordered lists, you can use the type attribute (1, A, a, I, i).

Common Mistakes

  • Putting text directly inside <ul> or <ol>: All content must be inside <li> elements. <ul>Text</ul> is invalid.
  • Using <br> to create list items: Each item needs its own <li>. Do not put multiple items in one <li> separated by <br>.
  • Incorrect nesting: A nested list must go inside an <li>, not directly inside another <ul>/<ol>.
  • Forgetting that <dl> is not for indentation: Do not use <dl> just to indent text. It is a semantic element for term-definition pairs.
