HTML Images

Beginner 45 min Module 7 of 14

Learning Objectives

  • Embed images using the <img> tag with src and alt attributes
  • Understand image formats: JPEG, PNG, GIF, SVG, WebP
  • Use <figure> and <figcaption> for semantic image grouping
  • Create responsive images with srcset and sizes

Images Explained

Think of it like this:

An <img> tag is like a picture frame on a wall. The frame (<img>) is mounted in a specific spot, and you slide a photo into it from behind. The src attribute is the reference to that photo — it tells the browser where to find the image file. The alt attribute is like a label next to the frame describing what the photo shows, so even if someone cannot see it, they understand what is there.

The <img> tag is a void element — it has no closing tag. It is also an inline element, but it behaves differently from text inline elements because it has a defined width and height. By default, an image displays at its original pixel size. You can control its dimensions using the width and height attributes, but always maintain the aspect ratio to avoid stretching.

Choosing the right image format matters: JPEG is best for photographs (millions of colors, small file size), PNG is best for graphics with transparency (logos, icons), GIF is for simple animations, SVG is for scalable vector graphics (crisp at any size), and WebP is a modern format that offers superior compression. Always include alt text — it is crucial for accessibility, appears when the image fails to load, and helps search engines understand your content.

Beginner Example

<img src="sunset.jpg" alt="A beautiful sunset over the ocean">
<img src="logo.png" alt="Company logo" width="200" height="100">
<img src="icon.svg" alt="Settings icon" width="32" height="32">

Line-by-Line Explanation:

  • Line 1: Basic image tag. src points to the image file. alt provides a text description — essential if the image does not load.
  • Line 2: Image with explicit width and height. Always include these to prevent layout shifts (Cumulative Layout Shift) as the page loads.
  • Line 3: SVG image. SVGs are vector-based and scale perfectly at any size. They are ideal for icons and logos.

Intermediate Example

<figure>
  <img src="architecture.jpg" alt="Modern building with glass facade">
  <figcaption>Downtown Arts Center, completed 2025</figcaption>
</figure>

<a href="full-image.jpg">
  <img src="thumbnail.jpg" alt="Click to view full-size landscape photo">
</a>

Line-by-Line Explanation:

  • Lines 1-4: <figure> groups the image with its caption <figcaption>. This is semantic HTML — it tells the browser that the caption belongs to this specific image. Screen readers announce them together.
  • Lines 6-8: An image wrapped in a link. Clicking the thumbnail opens the full-size image. This pattern is common in galleries and portfolios.

Advanced Example

<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 600px) 400px,
            (max-width: 1000px) 800px,
            1200px"
     alt="Mountain landscape at sunrise">

<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Product photo" loading="lazy">
</picture>

Line-by-Line Explanation:

  • Lines 1-5: srcset provides multiple image files at different widths. sizes tells the browser which width to use at different viewport sizes. The browser picks the best image — this makes your page faster on mobile.
  • Lines 8-12: The <picture> element allows format fallbacks. If the browser supports AVIF, it uses that. If not, it tries WebP. If neither works, it falls back to JPEG. This gives you modern formats while staying compatible.
  • Line 11: loading="lazy" tells the browser to only load the image when it is about to scroll into view. This dramatically improves initial page load time.

Interview Questions

Q: Why is the alt attribute important for accessibility?

Screen readers read the alt text aloud to visually impaired users, describing what the image shows. Without it, the screen reader might read the filename (e.g., "IMG_4592.jpg"), which is unhelpful. It also displays when the image fails to load and is used by search engines for image indexing.

Q: What is the difference between <img> and <picture>?

<img> displays a single image file. <picture> is a container that lets you specify multiple image sources with <source> elements, each with different formats or media conditions. The browser selects the best option. This enables format fallbacks and art direction (different crops for different screen sizes).

Q: How can you prevent layout shift caused by images loading?

Always include width and height attributes on your <img> tags. Modern browsers use these to calculate the aspect ratio and reserve the correct space before the image loads. Alternatively, use CSS aspect-ratio property. This prevents the content below from jumping down when the image loads, improving Core Web Vitals.

Common Mistakes

  • Missing alt text: Every image must have alt. For decorative images, use alt="" (empty) so screen readers skip them.
  • Not specifying width and height: Without dimensions, the browser does not know how much space to reserve, causing layout shifts as images load.
  • Using images at the wrong resolution: A 3000x2000 image displayed at 300x200 wastes bandwidth. Resize images to match their display size.
  • Broken image links: Double-check your src paths. A broken image shows an ugly broken icon. Always test your images locally.
