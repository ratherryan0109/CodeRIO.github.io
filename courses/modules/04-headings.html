HTML Headings

Beginner 40 min Module 4 of 14

Learning Objectives

  • Understand the six heading levels in HTML (h1 through h6)
  • Learn the correct hierarchy for headings in a document
  • Know how headings affect SEO and accessibility
  • Avoid common heading mistakes

Headings Explained

Think of it like this:

Headings in HTML are like the chapter structure of a textbook. The book title is the h1 (the most important). Chapter titles are h2. Section titles inside chapters are h3. Subsections are h4, and so on. Just like you would never put a chapter title as the book title, you should never skip heading levels or use them for their size instead of their meaning.

HTML provides six heading tags: <h1> through <h6>. <h1> is the most important and <h6> is the least. They are all block-level elements, meaning each one starts on a new line and takes the full width. Browsers automatically apply different font sizes to each level — h1 is the largest, h6 is the smallest — but you should never choose a heading based on size. Always choose based on meaning and hierarchy.

Headings play a critical role in SEO (search engines use them to understand your page structure) and accessibility (screen reader users navigate by jumping between headings). A well-structured page has exactly one h1 (the main title), followed by h2 elements for major sections, h3 for sub-sections, and so on. Skipping levels (going from h2 to h4) confuses both search engines and assistive technologies.

Visual Diagram

h1 — Page Title
h2 — Section Title
h3 — Sub-section Title
h4 — Sub-sub-section
h2 — Another Section

Always use headings in order. Do not skip levels.

Beginner Example

<h1>Welcome to My Website</h1>
<h2>About Me</h2>
<h3>My Background</h3>
<h3>My Hobbies</h3>
<h2>My Projects</h2>
<h3>Project 1</h3>
<h3>Project 2</h3>

Line-by-Line Explanation:

  • Line 1: <h1> — The main title of the page. There should be only one h1 per page. It tells search engines what the page is about.
  • Lines 2, 6: <h2> — Major sections under the main title. Each gets its own visual block.
  • Lines 3-5, 7-8: <h3> — Sub-sections inside each h2 topic. The hierarchy shows relationships between ideas.

Intermediate Example

<h1>The Complete Guide to Coffee</h1>
<h2>1. Coffee Types</h2>
<h3>Espresso</h3>
<h3>Latte</h3>
<h3>Cappuccino</h3>
<h2>2. Brewing Methods</h2>
<h3>French Press</h3>
<h4>Grind Size</h4>
<h4>Steeping Time</h4>
<h3>Pour Over</h3>
<h4>Water Temperature</h4>
<h2>3. Coffee Beans</h2>
<h3>Arabica vs Robusta</h3>

Line-by-Line Explanation:

  • Line 1: Single h1 for the entire guide. This is the book title of your page.
  • Lines 2, 7, 13: h2 elements for the three main chapters. Each chapter covers a distinct topic.
  • Lines 10-11: h4 inside h3 — correctly goes one level deeper without skipping. This shows that "Grind Size" and "Steeping Time" are details within "French Press."

Advanced Example

<header>
  <h1>Space Exploration News</h1>
  <p>Your daily dose of cosmos.</p>
</header>
<main>
  <article>
    <h2>Mars Rover Finds Ancient Water Traces</h2>
    <section>
      <h3>Key Discovery</h3>
      <p>Scientists found mineral deposits that form only in water.</p>
    </section>
    <section>
      <h3>What This Means</h3>
      <p>Evidence suggests Mars had liquid water 3 billion years ago.</p>
    </section>
  </article>
  <article>
    <h2>NASA Announces New Moon Mission</h2>
    <section>
      <h3>Mission Timeline</h3>
      <p>Launch is scheduled for 2027.</p>
    </section>
  </article>
</main>

Line-by-Line Explanation:

  • Line 2: The h1 is inside a <header> — a semantic element that groups introductory content.
  • Lines 7, 20: Each <article> has its own h2 as the article title. This is a common pattern: h1 for the page, h2 for each article.
  • Lines 9, 14: h3 elements inside <section> sub-divide each article. The heading hierarchy matches the content hierarchy — articles contain sections, so h2 contains h3.

Assignment

Create a "How-To Guide" page on any topic (e.g., "How to Bake a Cake" or "How to Change a Tire"). Use a single h1 for the guide title. Use h2 for each major step. Use h3 for sub-steps or details within each step. Include at least three h2 sections and two h3 elements per section. Write at least one paragraph under each heading. Save it as how-to-guide.html.

Hints:

  • Use an HTML validator (like validator.w3.org) to check that your heading hierarchy is correct.
  • Remember that headings are semantic — they describe structure, not font size. Use CSS to change sizes if needed.
  • If you find yourself wanting an h5 or h6, consider restructuring your content instead.

Interview Questions

Q: Can I use multiple h1 elements on a single page?

HTML5 allows multiple h1 elements if each is inside a separate sectioning root like <article> or <section>. However, for clarity and SEO best practices, it is recommended to use only one h1 per page as the main title.

Q: Should I choose a heading by its size or its meaning?

Always choose by meaning. Use CSS (font-size) to adjust the visual appearance if needed. Headings define the document outline — h1 is the most important, h6 is the least. Choosing by size only breaks the hierarchy and confuses screen readers.

Q: What happens if I skip a heading level (e.g., go from h2 to h4)?

Skipping heading levels creates a confusing document outline. Screen reader users rely on headings to navigate — jumping from h2 to h4 makes it seem like content is missing. It is a WCAG accessibility violation. Always maintain a logical hierarchy without gaps.

Common Mistakes

  • Using headings for their default size: Choosing h3 because "h2 is too big" is wrong. Use CSS to style the size and keep the heading semantic.
  • Skipping levels: Going from h1 directly to h4 breaks the outline. Always go h1 → h2 → h3 → h4 in order.
  • Multiple h1 on a page: While technically allowed in HTML5 within sectioning elements, it is best practice to have just one h1 per page for SEO clarity.
  • Empty headings: A heading with no content confuses screen readers. Every heading should have descriptive text inside it.
