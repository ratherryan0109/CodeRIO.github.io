SQL

SQL (Structured Query Language) is the universal language for managing and querying relational databases. Developed in the 1970s at IBM, SQL has become the industry standard for data storage, retrieval, and manipulation. Mastering SQL is essential for backend developers, data analysts, and anyone working with data-driven applications.

Overview

SQL was originally developed by Donald Chamberlin and Raymond Boyce at IBM in the early 1970s as SEQUEL (Structured English Query Language). It was later standardized by ANSI in 1986 and ISO in 1987. Today, SQL is supported by virtually every relational database management system (RDBMS) including PostgreSQL, MySQL, SQL Server, SQLite, and Oracle Database.

SQL operates on tables composed of rows and columns. It provides commands for data definition (DDL: CREATE, ALTER, DROP), data manipulation (DML: SELECT, INSERT, UPDATE, DELETE), data control (DCL: GRANT, REVOKE), and transaction management (TCL: BEGIN, COMMIT, ROLLBACK). Modern SQL extends to window functions, common table expressions (CTEs), and full-text search capabilities.

SQL underpins the data layer of most web applications, enterprise software, financial systems, and analytics platforms. With the rise of big data, SQL has evolved to work with distributed systems like Apache Spark, Google BigQuery, Amazon Redshift, and Snowflake. Even NoSQL databases like Cassandra and MongoDB offer SQL-like query interfaces.

Notes

1. Basic SELECT Query

The SELECT statement retrieves data from a table. You can select specific columns or all columns with *.

-- Select all columns
SELECT * FROM users;

-- Select specific columns
SELECT id, name, email FROM users;

-- With aliases
SELECT first_name || ' ' || last_name AS full_name
FROM employees;

-- Distinct values
SELECT DISTINCT city FROM customers;

2. WHERE Clause Filtering

Filter rows based on conditions using WHERE. Combine multiple conditions with AND/OR.

SELECT name, price, category
FROM products
WHERE price > 50
  AND category IN ('Electronics', 'Books')
  AND stock > 0
ORDER BY price DESC;

-- Pattern matching
SELECT * FROM users
WHERE email LIKE '%@gmail.com';

-- NULL checks
SELECT * FROM orders
WHERE shipped_date IS NULL;

3. JOIN Operations

JOINs combine rows from two or more tables based on related columns. Understanding JOINs is critical for working with normalized databases.

-- INNER JOIN: only matching rows
SELECT o.id, u.name, o.total
FROM orders o
INNER JOIN users u ON o.user_id = u.id;

-- LEFT JOIN: all from left, matching from right
SELECT u.name, COUNT(o.id) AS order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.name;

-- Self join: employees and their managers
SELECT e.name AS employee,
       m.name AS manager
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;

4. GROUP BY and Aggregation

Group rows and apply aggregate functions. HAVING filters groups after aggregation.

SELECT category,
       COUNT(*) AS product_count,
       AVG(price) AS avg_price,
       MAX(price) AS max_price,
       SUM(stock) AS total_stock
FROM products
GROUP BY category
HAVING COUNT(*) > 5
ORDER BY product_count DESC;

5. Subqueries

A subquery is a query nested inside another query. Can be used in SELECT, FROM, or WHERE clauses.

-- Subquery in WHERE
SELECT name, price
FROM products
WHERE price > (SELECT AVG(price) FROM products);

-- Correlated subquery
SELECT e.name, e.salary
FROM employees e
WHERE e.salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);

-- Subquery in FROM (derived table)
SELECT dept_name, avg_salary
FROM (
    SELECT d.name AS dept_name,
           AVG(e.salary) AS avg_salary
    FROM employees e
    JOIN departments d ON e.dept_id = d.id
    GROUP BY d.name
) AS dept_stats
WHERE avg_salary > 70000;

6. Window Functions

Window functions perform calculations across a set of rows related to the current row without collapsing them into a single output row.

SELECT name,
       department,
       salary,
       RANK() OVER (PARTITION BY department
                    ORDER BY salary DESC) AS dept_rank,
       LAG(salary) OVER (PARTITION BY department
                         ORDER BY salary) AS prev_salary,
       AVG(salary) OVER (PARTITION BY department) AS dept_avg
FROM employees;

7. Common Table Expressions (CTE)

CTEs provide a way to write named temporary result sets that can be referenced within a main query, improving readability and enabling recursion.

WITH high_value_customers AS (
    SELECT user_id, SUM(total) AS lifetime_value
    FROM orders
    GROUP BY user_id
    HAVING SUM(total) > 1000
)
SELECT u.name, u.email, hvc.lifetime_value
FROM users u
JOIN high_value_customers hvc ON u.id = hvc.user_id
ORDER BY hvc.lifetime_value DESC;

8. Indexes for Performance

Indexes speed up data retrieval at the cost of slower writes. Understanding when to create indexes is key to database performance.

-- Single column index
CREATE INDEX idx_users_email ON users(email);

-- Composite index (column order matters)
CREATE INDEX idx_orders_user_date
ON orders(user_id, order_date DESC);

-- Partial index
CREATE INDEX idx_active_users
ON users(email) WHERE active = true;

-- Explain query plan
EXPLAIN ANALYZE
SELECT * FROM users WHERE email = 'test@example.com';

Examples

Example 1: E-Commerce Database Queries

Common queries for an e-commerce application.

-- Top 10 customers by total spending
SELECT u.name, SUM(oi.quantity * oi.price) AS total_spent
FROM users u
JOIN orders o ON u.id = o.user_id
JOIN order_items oi ON o.id = oi.order_id
WHERE o.status = 'completed'
GROUP BY u.id, u.name
ORDER BY total_spent DESC
LIMIT 10;

-- Monthly sales report
SELECT DATE_TRUNC('month', o.order_date) AS month,
       COUNT(DISTINCT o.id) AS order_count,
       SUM(oi.quantity * oi.price) AS revenue
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
GROUP BY month
ORDER BY month DESC;

-- Products that need restocking (low stock with pending orders)
SELECT p.name, p.stock, COUNT(oi.id) AS pending_orders
FROM products p
LEFT JOIN order_items oi ON p.id = oi.product_id
JOIN orders o ON oi.order_id = o.id
WHERE o.status = 'pending'
  AND p.stock < 10
GROUP BY p.id, p.name, p.stock;

Example 2: Schema Design

Creating a normalized database schema for a blog platform.

CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(255) NOT NULL,
    body TEXT NOT NULL,
    published BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_posts_user_id ON posts(user_id);
CREATE INDEX idx_posts_created ON posts(created_at DESC);
