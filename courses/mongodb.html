MongoDB

MongoDB is a leading NoSQL document database that stores data in flexible, JSON-like documents. Developed by MongoDB Inc., it provides high performance, horizontal scalability, and a rich query language for modern applications.

Intermediate

Overview

MongoDB was created by Dwight Merriman and Eliot Horowitz in 2007 to solve scalability issues with relational databases. It uses a document model where data is stored as BSON (Binary JSON) documents in collections, rather than rows in tables. This schema-less design allows for flexible, evolvable data structures that can change over time without migrations.

MongoDB's document model maps naturally to object-oriented programming, reducing the need for ORMs and JOINs. Documents can contain nested objects, arrays, and even embedded documents, making it ideal for hierarchical data. The database supports rich queries including field queries, range queries, text search, geospatial queries, and aggregation pipelines.

Key features include replica sets for high availability (automatic failover), sharding for horizontal scaling, change streams for real-time data, transactions for multi-document atomicity (since v4.0), and Atlas for fully managed cloud deployments. MongoDB is used by companies like Uber, eBay, Google, and Cisco for applications requiring flexible data models and scale.

Why Learn MongoDB?

Flexible Document Model

Store data as JSON documents with dynamic schemas. Add fields on the fly without migrations. Perfect for agile development and evolving requirements.

High Performance

In-memory processing, indexed queries, and horizontal scaling provide sub-millisecond response times for read-heavy workloads and real-time applications.

Horizontal Scalability

Sharding distributes data across multiple servers automatically. Scale out by adding more machines, not by buying bigger hardware (vertical scaling).

Rich Ecosystem & Community

Mongoose ODM, Compass GUI, Atlas cloud, drivers for every language, and extensive documentation make MongoDB accessible.

Notes

1. CRUD Operations in MongoDB Shell

MongoDB Shell (mongosh) is the CLI for interacting with MongoDB. Use it for prototyping, administration, and debugging. CRUD operations use collection methods that return a cursor or document.

// Create
db.users.insertOne({ name: 'Alice', email: 'alice@test.com', age: 30 })
db.users.insertMany([{ name: 'Bob', age: 25 }, { name: 'Charlie', age: 35 }])

// Read
db.users.find({ age: { $gte: 25 } })
db.users.findOne({ email: 'alice@test.com' })
db.users.find().sort({ age: -1 }).limit(10).skip(20)

// Update
db.users.updateOne({ name: 'Alice' }, { $set: { age: 31 } })
db.users.updateMany({}, { $inc: { loginCount: 1 } })

// Delete
db.users.deleteOne({ name: 'Charlie' })
db.users.deleteMany({ age: { $lt: 18 } })

2. Aggregation Pipeline

The aggregation pipeline processes documents through a sequence of stages. Each stage transforms the documents as they pass through. It's powerful for analytics, reporting, and data transformation.

db.orders.aggregate([
  // Stage 1: Filter
  { $match: { status: 'completed' } },

  // Stage 2: Group by customer, calculate totals
  { $group: {
    _id: '$customerId',
    totalSpent: { $sum: '$amount' },
    orderCount: { $sum: 1 },
    avgOrder: { $avg: '$amount' }
  }},

  // Stage 3: Sort by total spent descending
  { $sort: { totalSpent: -1 } },

  // Stage 4: Limit to top 10
  { $limit: 10 },

  // Stage 5: Lookup customer details
  { $lookup: {
    from: 'customers',
    localField: '_id',
    foreignField: '_id',
    as: 'customer'
  }}
])

3. Indexing Strategy

Indexes dramatically improve query performance by reducing the number of documents scanned. Use explain() to check if queries use indexes. Choose index fields based on query patterns, sort order, and selectivity.

// Create indexes
db.users.createIndex({ email: 1 }, { unique: true })
db.orders.createIndex({ customerId: 1, createdAt: -1 })
db.posts.createIndex({ title: 'text', content: 'text' })
db.locations.createIndex({ coordinates: '2dsphere' })

// Compound index rules:
// 1. Equality fields first
// 2. Sort fields next
// 3. Range fields last
db.orders.createIndex({ status: 1, createdAt: -1, total: 1 })

// Check query execution
db.users.find({ email: 'test@test.com' }).explain('executionStats')

// Drop unused indexes
db.users.dropIndex('email_1')

4. Mongoose Schema & Models

Mongoose is the most popular MongoDB ODM for Node.js. It provides a schema-based solution with built-in validation, casting, middleware, and population for referencing documents.

const userSchema = new mongoose.Schema({
  name:     { type: String, required: true, trim: true },
  email:    { type: String, required: true, unique: true, lowercase: true },
  age:      { type: Number, min: 13, max: 120 },
  role:     { type: String, enum: ['user', 'admin'], default: 'user' },
  address: {
    street: String,
    city:   String,
    zip:    String
  },
  tags:     [String],
  createdAt: { type: Date, default: Date.now }
}, { timestamps: true });

// Virtual property
userSchema.virtual('isAdult').get(function() {
  return this.age >= 18;
});

// Pre-save middleware
userSchema.pre('save', async function(next) {
  if (this.isModified('password')) {
    this.password = await bcrypt.hash(this.password, 10);
  }
  next();
});

module.exports = mongoose.model('User', userSchema);

5. Data Modeling: Embedding vs Referencing

Choose embedding when data is always accessed together, has one-to-few relationships, and doesn't grow unbounded. Choose referencing when data is large, shared across documents, or has one-to-many/many-to-many relationships.

// Embedded (good for address in user)
{
  _id: ObjectId('...'),
  name: 'Alice',
  addresses: [
    { street: '123 Main', city: 'NYC', type: 'home' },
    { street: '456 Oak',  city: 'LA',  type: 'work' }
  ]
}

// Referenced (good for orders in user)
// User document
{ _id: ObjectId('user1'), name: 'Alice' }

// Order documents
{ _id: ObjectId('order1'), userId: 'user1', total: 100 }
{ _id: ObjectId('order2'), userId: 'user1', total: 200 }

// In Mongoose, use populate to join
const orders = await Order.find({ userId: 'user1' }).populate('items.productId')

6. Transactions

MongoDB supports multi-document ACID transactions since v4.0. Use transactions for operations that require atomicity across multiple documents or collections. Always handle transaction errors and retry logic.

async function transferMoney(fromId, toId, amount) {
  const session = await mongoose.startSession();
  session.startTransaction();

  try {
    const fromAccount = await Account.findById(fromId).session(session);
    const toAccount = await Account.findById(toId).session(session);

    if (fromAccount.balance < amount) {
      throw new Error('Insufficient funds');
    }

    fromAccount.balance -= amount;
    toAccount.balance += amount;

    await fromAccount.save({ session });
    await toAccount.save({ session });

    await session.commitTransaction();
    console.log('Transfer successful');
  } catch (error) {
    await session.abortTransaction();
    console.error('Transfer failed:', error);
  } finally {
    session.endSession();
  }
}

7. Change Streams

Change streams allow applications to watch real-time changes in collections, databases, or deployments. They use the oplog (operations log) and provide resume tokens for fault-tolerant processing.

// Watch all insert operations on orders collection
const changeStream = db.collection('orders').watch([
  { $match: { operationType: 'insert' } }
]);

changeStream.on('change', (change) => {
  console.log('New order:', change.fullDocument);
  // Send notification, update cache, trigger workflow
});

// Resume from a specific point
const resumeToken = changeStream.resumeToken;
// Store resume token for recovery
const newStream = db.collection('orders').watch([], {
  resumeAfter: resumeToken
});

// In Node.js with Mongoose
const pipeline = [{ $match: { operationType: 'update' } }];
const stream = Order.watch(pipeline);
stream.on('change', (change) => console.log('Order updated:', change));

8. Replica Sets

A replica set is a group of MongoDB servers that maintain the same data set. One primary accepts writes, secondaries replicate data. If the primary fails, an election promotes a secondary. This provides high availability and data redundancy.

// Start mongod instances for replica set
mongod --replSet rs0 --port 27017 --dbpath /data/rs0-1
mongod --replSet rs0 --port 27018 --dbpath /data/rs0-2
mongod --replSet rs0 --port 27019 --dbpath /data/rs0-3

// Initiate replica set
rs.initiate({
  _id: 'rs0',
  members: [
    { _id: 0, host: 'localhost:27017' },
    { _id: 1, host: 'localhost:27018' },
    { _id: 2, host: 'localhost:27019', arbiterOnly: true }
  ]
})

// Check replica set status
rs.status()

// Connection string for replica set
mongoose.connect('mongodb://localhost:27017,localhost:27018,localhost:27019/mydb?replicaSet=rs0')

Example 1: E-commerce Product Catalog API

const productSchema = new mongoose.Schema({
  name:        { type: String, required: true },
  slug:        { type: String, unique: true },
  description: String,
  price:       { type: Number, required: true, min: 0 },
  category:    { type: mongoose.Schema.Types.ObjectId, ref: 'Category' },
  tags:        [String],
  variants:    [{
    name:   String,
    sku:    { type: String, required: true },
    price:  Number,
    stock:  { type: Number, default: 0 }
  }],
  ratings: {
    average: { type: Number, default: 0 },
    count:   { type: Number, default: 0 }
  },
  isActive:    { type: Boolean, default: true }
}, { timestamps: true });

// Text search index
productSchema.index({ name: 'text', description: 'text' });
// Compound index for category queries
productSchema.index({ category: 1, price: -1 });

// Find active products in a category with pagination
app.get('/api/products', async (req, res) => {
  const { category, minPrice, maxPrice, page = 1, limit = 20 } = req.query;
  const filter = { isActive: true };

  if (category) filter.category = category;
  if (minPrice || maxPrice) {
    filter.price = {};
    if (minPrice) filter.price.$gte = Number(minPrice);
    if (maxPrice) filter.price.$lte = Number(maxPrice);
  }

  const products = await Product.find(filter)
    .populate('category', 'name')
    .sort({ createdAt: -1 })
    .skip((page - 1) * limit)
    .limit(Number(limit));

  const total = await Product.countDocuments(filter);
  res.json({ products, total, page, totalPages: Math.ceil(total / limit) });
});

Example 2: Aggregation for Sales Dashboard

db.sales.aggregate([
  { $match: { date: { $gte: ISODate('2024-01-01'), $lt: ISODate('2025-01-01') } } },

  // Group by month
  { $group: {
    _id: { $dateToString: { format: '%Y-%m', date: '$date' } },
    totalSales: { $sum: '$amount' },
    orderCount: { $sum: 1 },
    avgOrderValue: { $avg: '$amount' }
  }},

  { $sort: { _id: 1 } },

  // Calculate running total
  { $setWindowFields: {
    sortBy: { _id: 1 },
    output: {
      runningTotal: { $sum: '$totalSales', window: { documents: ['unbounded', 'current'] } }
    }
  }}
])
