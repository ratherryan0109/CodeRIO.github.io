TypeScript

TypeScript is a typed superset of JavaScript developed by Microsoft that compiles to plain JavaScript. It adds optional static typing, classes, interfaces, generics, and advanced tooling to catch errors before runtime.

Intermediate

Overview

TypeScript was created by Anders Hejlsberg at Microsoft in 2012 to address the challenges of building large-scale JavaScript applications. It adds a powerful type system on top of JavaScript, enabling developers to catch type-related bugs at compile time rather than runtime. TypeScript code compiles to clean, readable JavaScript that runs anywhere JS runs.

The language supports all JavaScript syntax plus type annotations, interfaces, enums, tuples, generics, decorators, and advanced utility types. TypeScript's type system is structural (duck typing) rather than nominal, meaning compatibility is based on shape rather than explicit declarations. The compiler also supports modern ECMAScript features and downlevels them to older JS versions.

TypeScript has become the standard for professional JavaScript development. Major frameworks like React, Angular, Vue, and Node.js all have first-class TypeScript support. The type checker provides intelligent autocompletion, refactoring tools, and inline documentation in editors like VS Code, significantly improving developer productivity.

Why Learn TypeScript?

Catch Errors Early

Type checking at compile time catches bugs before they reach production. Null pointer exceptions, undefined property access, and type mismatches are caught instantly.

Better Tooling & IDE Support

TypeScript enables rich autocompletion, go-to-definition, find references, and safe refactoring. VS Code, WebStorm, and other editors provide first-class TS support.

Self-Documenting Code

Type annotations serve as live documentation that never goes out of date. Interfaces and types clearly define the shape of data flowing through your application.

Industry Standard

TypeScript is used by 78% of professional JavaScript developers. It is required for Angular, recommended for React, and used by most enterprise JS projects.

Notes

1. Basic Type Annotations

Type annotations follow a colon syntax after variable names. TypeScript can also infer types automatically from the assigned value. Use type annotations when inference is not possible or when you want explicit documentation.

let name: string = 'Alice';
let age: number = 30;
let isActive: boolean = true;

// Type inference - TS knows this is a string
let message = 'Hello';  // inferred as string

// Union types
let id: string | number = 'abc123';
id = 456;  // valid

2. Interfaces vs Type Aliases

Interfaces and type aliases are similar but have key differences. Interfaces can be extended and merged, while type aliases can represent unions, intersections, and primitive types. Prefer interfaces for object shapes and types for complex compositions.

// Interface - extends, implements, declaration merging
interface User {
  id: number;
  name: string;
  email: string;
}
interface Admin extends User {
  role: 'admin';
}

// Type alias - unions, intersections, primitives
type Status = 'active' | 'inactive' | 'pending';
type Result = { success: true; data: unknown }
            | { success: false; error: string };

3. Generics

Generics allow creating reusable components that work with multiple types while preserving type safety. They act as type variables that capture the type passed in. Generics can be constrained with extends to limit acceptable types.

function identity(arg: T): T {
  return arg;
}

const num = identity(42);
const str = identity('hello');  // inferred

// Generic constraint
function getLength(item: T): number {
  return item.length;
}

// Generic interface
interface Response {
  status: number;
  data: T;
  message: string;
}

4. Type Guards & Narrowing

Type narrowing is the process of refining a broader type to a more specific type within conditional blocks. TypeScript uses typeof, instanceof, discriminated unions, and custom type predicates for narrowing.

function format(input: string | number) {
  if (typeof input === 'string') {
    return input.toUpperCase();  // TS knows it's string
  }
  return input.toFixed(2);  // TS knows it's number
}

// Discriminated union
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'square'; side: number };

function area(shape: Shape): number {
  if (shape.kind === 'circle') {
    return Math.PI * shape.radius ** 2;
  }
  return shape.side ** 2;
}

5. Mapped Types

Mapped types allow transforming every property of an existing type using the in keyof syntax. They are the foundation for utility types like Partial, Readonly, and Pick. Combined with key remapping and as clauses.

type Readonly = {
  readonly [K in keyof T]: T[K];
};

type Optional = {
  [K in keyof T]?: T[K];
};

// Key remapping with as
type Getters = {
  [K in keyof T as `get${Capitalize}`]: () => T[K];
};

interface Person { name: string; age: number; }
// Result: { getName: () => string; getAge: () => number; }

6. Conditional Types & infer

Conditional types select between two types based on a condition using the extends ? : syntax. The infer keyword extracts types from within other types, enabling powerful type-level programming.

type IsString = T extends string ? 'yes' : 'no';
type A = IsString<'hello'>;  // 'yes'
type B = IsString<42>;       // 'no'

// infer extracts the element type from an array
type Unpack = T extends Array ? U : T;
type Item = Unpack;  // string

// ReturnType built-in uses infer
type ReturnType =
  T extends (...args: any[]) => infer R ? R : any;

7. Template Literal Types

Template literal types use string literal interpolation to create complex string types. They can combine unions of string literals and are useful for event names, CSS properties, and API route patterns.

type Color = 'red' | 'green' | 'blue';
type Variant = 'light' | 'dark';
type ColorVariant = `${Color}-${Variant}`;
// 'red-light' | 'red-dark' | 'green-light' | ...

type EventName = `on${Capitalize}`;
// 'onClick' | 'onChange' | 'onSubmit' | ...

// Intrinsic string types
type Upper = Uppercase<'hello'>;  // 'HELLO'
type Lower = Lowercase<'WORLD'>;  // 'world'

8. Declaration Files & Ambient Types

Declaration files (.d.ts) describe the shape of JavaScript modules for TypeScript. They allow TS to understand libraries without source code. Ambient declarations use declare to describe values that exist at runtime.

// globals.d.ts
declare const API_KEY: string;
declare function fetchData(url: string): Promise;

// Module declaration for untyped JS library
declare module 'my-library' {
  export function doSomething(input: string): number;
  export const version: string;
}

// Augment existing module
declare module 'express' {
  interface Request {
    user?: { id: number; name: string };
  }
}

Examples

Example 1: Type-safe API Client

interface ApiConfig {
  baseUrl: string;
  timeout?: number;
  headers?: Record;
}

interface ApiResponse {
  data: T;
  status: number;
  message: string;
}

class ApiClient {
  constructor(private config: ApiConfig) {}

  async get(path: string): Promise> {
    const response = await fetch(`${this.config.baseUrl}${path}`, {
      headers: this.config.headers,
      signal: AbortSignal.timeout(this.config.timeout ?? 5000)
    });
    return response.json();
  }
}

interface User {
  id: number;
  name: string;
  email: string;
}

const api = new ApiClient({ baseUrl: 'https://api.example.com' });
const user = await api.get('/users/1');
console.log(user.data.name);  // type-safe access

Example 2: Type-safe Event Emitter

type EventMap = {
  userLogin: { userId: number; timestamp: Date };
  pageView: { path: string; referrer: string | null };
  error: { message: string; code: number };
};

class TypedEmitter> {
  private handlers = new Map>();

  on(
    event: K,
    handler: (data: T[K]) => void
  ): void {
    if (!this.handlers.has(event)) {
      this.handlers.set(event, new Set());
    }
    this.handlers.get(event)!.add(handler);
  }

  emit(event: K, data: T[K]): void {
    this.handlers.get(event)?.forEach(handler => handler(data));
  }
}

const emitter = new TypedEmitter();
emitter.on('userLogin', data => {
  console.log(data.userId);  // type-safe
});
