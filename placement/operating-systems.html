Detailed Notes

Process vs Thread

Process: independent execution unit with own memory space. Thread: light-weight process sharing memory within a process. Multi-threading improves responsiveness and resource utilization.

CPU Scheduling Algorithms

FCFS (non-preemptive, convoy effect), SJF (optimal avg waiting time, starvation), Round Robin (time quantum, fair), Priority (starvation, aging).

Deadlock Conditions

Mutual exclusion, hold and wait, no preemption, circular wait. Prevention: break any condition. Avoidance: Banker's algorithm. Detection: resource allocation graph.

Page Replacement Algorithms

FIFO (Belady's anomaly), LRU (optimal but costly), Optimal (theoretical perfect), Clock (approximation of LRU). Minimize page faults.

Disk Scheduling

FCFS (fair but slow), SCAN (elevator algorithm), C-SCAN (uniform wait time), LOOK (goes only as far as requests). Seek time optimization.

Virtual Memory

Allows execution of processes larger than physical memory. Demand paging, page faults, swapping. Thrashing occurs when system spends more time paging than executing.

Interview Questions

What is the difference between a mutex and a semaphore?

Mutex: binary lock for mutual exclusion, owned by thread that acquires it. Semaphore: counting mechanism, can signal/wait, not owned. Binary semaphore != mutex (no ownership).

Explain the Banker's algorithm.

Deadlock avoidance algorithm. Processes declare max resource needs. System checks if granting a request leads to a safe state (some sequence where all processes can finish).

What is thrashing and how to handle it?

Excessive paging due to low memory. System spends time swapping pages instead of executing. Solutions: increase RAM, reduce degree of multiprogramming, working set model.

What is the difference between paging and segmentation?

Paging: fixed-size frames, no external fragmentation, internal fragmentation possible. Segmentation: variable-size logical segments, external fragmentation, matches user's view of memory.

Explain the Dining Philosophers problem.

Classic synchronization problem. 5 philosophers, 5 forks. Solution: pick up both forks only if available, use mutex/semaphore, or limit concurrent philosophers to 4.
