Computer Networks

Networking fundamentals for interviews — OSI model, TCP/IP, protocols, and network security concepts.

5 Modules 20+ Questions High Priority

OSI Model

7-layer model: Physical, Data Link, Network, Transport, Session, Presentation, Application. Each layer handles specific networking functions.

Layer 7: HTTP/FTP/SMTP
Layer 4: TCP/UDP
Layer 3: IP/ICMP

TCP/IP Suite

4-layer model: Network Interface, Internet, Transport, Application. TCP (connection-oriented, reliable) vs UDP (connectionless, faster).

TCP: 3-way handshake
SYN -> SYN-ACK -> ACK

Network Protocols

HTTP/HTTPS, DNS (port 53), DHCP (ports 67/68), FTP (21), SMTP (25), IMAP (143), ARP, ICMP, SNMP. Each serves a distinct purpose.

DNS: domain -> IP
ARP: IP -> MAC address

Network Security

Firewalls, IDS/IPS, VPNs, TLS/SSL, encryption (symmetric/asymmetric), digital signatures, certificates, and common attacks (DDoS, MITM).

TLS handshake: cipher suite
negotiation + cert verification

Detailed Notes

TCP Features

Flow control (sliding window), congestion control (slow start, AIMD, fast retransmit), error detection (checksum), sequencing, and retransmission. Reliable data delivery.

IPv4 vs IPv6

IPv4: 32-bit (4.3B addresses), dotted decimal, NAT needed. IPv6: 128-bit, hex notation, built-in security (IPsec), no NAT required. Transition via dual stack and tunneling.

Subnetting & CIDR

Subnet mask divides network/host bits. CIDR notation (e.g., /24). VLSM allows variable subnet masks. Supernetting combines networks. Reduces routing table size.

Routing Protocols

RIP (distance vector, hop count), OSPF (link-state, Dijkstra), BGP (path vector, internet backbone). Interior vs exterior gateway protocols.

HTTP Methods & Status Codes

GET (read), POST (create), PUT (update), DELETE, PATCH. 1xx info, 2xx success, 3xx redirect, 4xx client error, 5xx server error. Common: 200, 301, 400, 401, 403, 404, 500.

Switching Techniques

Circuit switching (telephone), packet switching (internet), message switching. Virtual circuit vs datagram. Store-and-forward vs cut-through switching.

Interview Questions

What happens when you type a URL in a browser?

DNS lookup -> TCP connection (3-way handshake) -> HTTP request -> server processes -> HTTP response -> browser renders HTML/CSS/JS. Includes ARP, routing, and load balancers.

Explain the difference between TCP and UDP.

TCP: connection-oriented, reliable, ordered, flow/congestion control, slower. UDP: connectionless, no guarantees, faster, lower overhead. TCP for web/email, UDP for streaming/DNS.

What is a subnet mask and how does it work?

A 32-bit mask that separates IP into network and host portions. AND operation with IP yields network address. Used for routing and determining if destination is local/remote.

Explain the concept of NAT.

Network Address Translation maps private IPs to a public IP. Enables multiple devices on a LAN to share one public IP. Types: static, dynamic, PAT (port address translation).

What is the difference between a switch and a router?

Switch: Layer 2 device, uses MAC addresses, connects devices within same network. Router: Layer 3 device, uses IP addresses, connects different networks, performs routing.
