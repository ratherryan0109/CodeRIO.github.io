Database Management Systems

Core DBMS concepts for placement interviews — SQL, normalization, transactions, and query optimization.

4 Modules 25+ Questions High Priority

Normalization

Process of organizing data to reduce redundancy. 1NF, 2NF, 3NF, BCNF. Higher normal forms eliminate anomalies and ensure data integrity.

1NF: Atomic values
2NF: No partial dependency
3NF: No transitive dependency

SQL

DDL (CREATE, ALTER, DROP), DML (SELECT, INSERT, UPDATE, DELETE), DCL (GRANT, REVOKE), and TCL (COMMIT, ROLLBACK).

SELECT * FROM Employees
WHERE salary > 50000
ORDER BY name DESC;

Transactions

ACID properties — Atomicity, Consistency, Isolation, Durability. Concurrency control, locks, deadlocks, and isolation levels.

BEGIN TRANSACTION;
UPDATE accounts SET bal = bal - 100;
COMMIT;

Indexing

B-Tree, Hash indexes, clustered vs non-clustered. Speeds up read queries but slows writes. Use on frequently queried columns.

CREATE INDEX idx_name
ON Employees(last_name);

Detailed Notes

ER Model & Relational Model

ER diagrams use entities, attributes, and relationships. Relational model maps to tables. Keys: primary key (unique identifier), foreign key (references another table), candidate key, super key.

Join Types

INNER JOIN (matching rows), LEFT/RIGHT JOIN (all from one side), FULL OUTER JOIN (all rows), CROSS JOIN (Cartesian product), SELF JOIN (table with itself).

ACID Properties

Atomicity: all or nothing. Consistency: valid state. Isolation: concurrent transactions don't interfere. Durability: committed changes persist.

Isolation Levels

Read Uncommitted (dirty reads), Read Committed (no dirty reads), Repeatable Read (no non-repeatable reads), Serializable (highest isolation).

Functional Dependencies

X -> Y means X determines Y. Used in normalization. Armstrong's axioms: reflexivity, augmentation, transitivity. Closure of attributes helps find candidate keys.

Query Optimization

Query parsing, rewriting, cost-based optimization. Use EXPLAIN plans, proper indexes, avoid SELECT *, use JOINs instead of subqueries where possible.

Interview Questions

What is the difference between DELETE, TRUNCATE, and DROP?

DELETE: DML, removes rows, can have WHERE, slower, triggers fire. TRUNCATE: DDL, removes all rows, faster, no triggers. DROP: DDL, removes entire table structure.

Explain the difference between clustered and non-clustered index.

Clustered: determines physical order of data, one per table. Non-clustered: separate structure with pointers to data rows, multiple allowed.

What is a deadlock and how to avoid it?

Two or more transactions waiting for each other's locks. Solutions: lock ordering, timeout, deadlock detection with rollback, using lock hints.

Explain normalization and denormalization.

Normalization reduces redundancy (up to 3NF/BCNF). Denormalization adds redundancy for read performance. Used in data warehouses and reporting.

What is a view? What are its advantages?

A virtual table based on a query. Advantages: security (hide sensitive columns), simplify complex queries, data abstraction, logical data independence.
